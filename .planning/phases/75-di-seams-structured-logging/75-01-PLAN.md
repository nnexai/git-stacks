<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workspace YAML hooks -> subprocess launch seam | Untrusted persisted hook commands cross into a mutable `_exec.spawn` boundary. |
| Workspace-git orchestration -> git helper seam | Module orchestration crosses into subprocess-owning git helpers through a mutable exported object. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-75-01 | T | `src/lib/workspace-lifecycle.ts` `_exec.spawn` seam | mitigate | Reuse the exact `SpawnHandle` contract from `src/lib/lifecycle.ts`, keep command/env assembly unchanged, and lock the call shape with seam tests so the new seam cannot silently alter hook execution semantics. |
| T-75-02 | T | `src/lib/workspace-git.ts` `_exec` forwarding | mitigate | Forward to the existing `src/lib/git.ts` helpers only, do not edit `git.ts`, and add tests that assert the exact repo path, branch, and options passed through `_exec.pushBranch`, `_exec.fetchOrigin`, and `_exec.rebaseBranch`. |
| T-75-03 | D | `src/lib/workspace-lifecycle.ts` captured hook reader | mitigate | Keep the captured helper's line-by-line reader behavior identical to `src/lib/lifecycle.ts` and cover it with `captured: true` tests so replacing the seam cannot deadlock or drop output. |
</threat_model>
