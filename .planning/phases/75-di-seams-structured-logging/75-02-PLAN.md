<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Process env -> observability bootstrap | Untrusted environment variables control whether debug output is enabled and which categories are emitted. |
| Internal debug metadata -> stderr sink | Operational details cross from internal modules into a user-visible stderr stream while stdout contracts must remain stable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-75-04 | I | `src/lib/observability.ts` stderr formatter | mitigate | Keep the existing stderr-only sink, add structured-field rendering only in the formatter, and lock JSON-safe stdout with `tests/commands/status-json.test.ts` so debug data never leaks into stdout. |
| T-75-05 | T | `src/index.ts` env bootstrap | mitigate | Parse `GS_DEBUG` and the `GIT_STACKS_DEBUG=1` alias only once in `src/index.ts`, and reject distributed env parsing elsewhere so selector behavior cannot drift by command. |
| T-75-06 | D | `src/lib/observability.ts` selector filtering | mitigate | Normalize selector tokens centrally with a fixed alias map and cover positive and negative filter cases in both unit and CLI tests so a bad selector cannot unexpectedly enable every module. |
</threat_model>
