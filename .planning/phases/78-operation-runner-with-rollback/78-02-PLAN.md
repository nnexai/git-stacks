**Key constraint:** The current file-ops API does not return the list of created paths, so the undo for file ops cannot delete specific files individually. Two acceptable strategies (executor picks):

  Strategy A (recommended — simpler, matches D-08 spirit): The undo for file ops is `() => {}` — a no-op. Rationale: when the worktree is already being torn down by an earlier undo (the worktree dir is deleted by `removeWorktree`), every file inside it goes with it. For workspace-instance file ops, the undo deletes the workspace dir entry only if it's empty, otherwise no-op. This is honest about the runner's tracking granularity and matches the "best-effort" spirit of ENGN-03.

  Strategy B (more thorough — requires API change): Extend `applyFileOpsForRepo`/`applyFileOpsForWorkspace` to return `{ ok, createdPaths: string[] }` and have the undo iterate `createdPaths` calling `rmSync`. This is a larger change and pulls files.ts into the phase scope.

  **DECISION: Use Strategy A.** Rationale: per-repo file ops always land inside `wsRepo.task_path`, which is removed by the worktree undo (see D-08). Workspace-instance file ops land inside `wsDir = join(tasksDir, wsName)`, which the runner does NOT remove (the wsDir stays so the user can inspect the failure). An empty-dir cleanup of `wsDir` after worktree rollbacks is a reasonable "best-effort" undo. This keeps files.ts unchanged.

- **D-09:** NOT tracked: `pre_create` and `post_create` hooks. They run inside the orchestrating try block so their failure triggers rollback of preceding tracked steps, but the hooks themselves are not pushed onto the compensation stack.
- **D-10:** `writeWorkspace()` is the commit point — not a tracked step. The runner completes the tracked sequence first, the orchestrating try block exits cleanly, runner.result() returns `{ ok: true }`, AND THEN createWorkspace calls writeWorkspace once. Success criterion 1 is satisfied by construction: the YAML write is unreachable if any prior step threw.
- **D-11:** `runIntegrationGenerate()` runs AFTER writeWorkspace. Its failures are logged through `onProgress` only and do not trigger rollback — the workspace is already committed.
- **D-12:** Final ordering top-to-bottom inside `createWorkspace()`:
  1. `pre_create` hooks (not tracked; failure throws → rollback no-op since stack is empty)
  2. Worktree creation per repo (tracked)
  3. `ensureUpstreamTracking` across all worktree repos (NOT tracked; best-effort, already tolerant — wrap in try/catch with onProgress warning, don't fail creation)
  4. Per-repo file ops (tracked)
  5. Workspace file ops (tracked)
  6. Env-file writes (tracked)
  7. `post_create` hooks (not tracked; failure throws → rollback all tracked steps above)
  8. `writeWorkspace()` — commit point, OUTSIDE the runner stack
  9. `runIntegrationGenerate()` — post-commit, failures logged only
- **D-13:** `post_create` hook failures still abort creation. The only behavior change vs today is that on abort, already-created worktrees and file-op artifacts are cleaned up.
- **D-14:** Rollback messages flow through `onProgress`. NO `console.error`. The runner from Plan 01 already enforces this.
- **D-15:** `Rollback: ...` and `Rollback error: ...` prefixes (already enforced by Plan 01's runner — createWorkspace just supplies the step `name` to `runner.do(name, ...)` and the runner builds the prefix).

```typescript
export async function createWorkspace(
  inputs: CreateWorkspaceInputs,
  onProgress?: ProgressCallback
): Promise<CreateWorkspaceResult> {
  return timeOperation(OBS_CATEGORY, "createWorkspace", async () => {
    const config = readGlobalConfig()
    const tasksDir = getTasksDir(config.workspace_root)
    const wsDir = join(tasksDir, inputs.wsName)
    const worktreeRepos = inputs.repos.filter(isWorktreeRepo)

    // Ensure the wsDir parent exists before tracked steps push artifacts into it.
    // The wsDir itself is created on demand by the worktree forwards, but mkdir -p
    // is needed for nested workspace names.
    if (!existsSync(wsDir)) mkdirSync(wsDir, { recursive: true })

    const baseEnv: Record<string, string> = {
      GS_WORKSPACE_NAME: inputs.wsName,
      GS_WORKSPACE_BRANCH: inputs.branch,
      GS_WORKSPACE_PATH: tasksDir,
      GS_TRIGGERED_BY: "create",
      ...(inputs.wsEnv ?? {}),
    }

    // ─── Build the workspace object early so file ops and env writes have a Workspace
    // to operate on. This object is held in memory until the commit point (D-10) — it is
    // ONLY written to disk after the runner returns ok.
    const settingsIntegrations =
      inputs.wsIntegrationSettings && Object.keys(inputs.wsIntegrationSettings).length > 0
        ? { settings: { integrations: inputs.wsIntegrationSettings } }
        : {}
    const workspaceObj: Workspace = {
      name: inputs.wsName,
      schema_version: "1",
      description: inputs.description || undefined,
      branch: inputs.branch,
      created: new Date().toISOString().split("T")[0]!,
      ...(inputs.templateName ? { template: inputs.templateName } : {}),
      ...(inputs.wsHooks ? { hooks: inputs.wsHooks } : {}),
      repos: inputs.repos,
      ...(inputs.wsEnv ? { env: inputs.wsEnv } : {}),
      ...(inputs.wsEnvFile ? { env_file: inputs.wsEnvFile } : {}),
      ...(inputs.wsFiles ? { files: inputs.wsFiles } : {}),
      ...settingsIntegrations,
      ...(inputs.wsPorts ? { ports: inputs.wsPorts } : {}),
      ...((inputs.labels?.length ?? 0) > 0 ? { labels: inputs.labels } : {}),
    } as Workspace

    const runner = createRunner(onProgress)

    try {
      // ─── D-12 step 1: pre_create hooks (NOT tracked, D-09) ──────────────────────
      if (inputs.wsHooks?.pre_create?.length) {
        await runWorkspaceHooks(inputs.wsHooks.pre_create, tasksDir, baseEnv)
      }

      // ─── D-12 step 2: worktree creation (TRACKED) ──────────────────────────────
      for (const repo of worktreeRepos) {
        await runner.do(
          `create worktree ${repo.name}`,
          async () => {
            onProgress?.(`Creating worktree for ${repo.name}`)
            await createWorktree(repo.main_path, repo.task_path, inputs.branch)
            onProgress?.(`created worktree for ${repo.name}`)
          },
          async () => {
            // The runner's per-undo try/catch will convert any throw here into a
            // "Rollback error: create worktree <name> failed (...)" message.
            await removeWorktree(repo.main_path, repo.task_path)
          },
        )
      }

      // ─── D-12 step 3: ensureUpstreamTracking (NOT tracked, best-effort) ────────
      // Wrapped in try/catch so a tracking failure does not abort creation.
      try {
        const trackingResults = await Promise.all(
          worktreeRepos.map(repo => ensureUpstreamTracking(repo.main_path, inputs.branch))
        )
        const tracked = trackingResults.filter(r => r.tracked).length
        if (tracked > 0) onProgress?.(`Upstream tracking set for ${tracked} repo(s)`)
      } catch (err) {
        onProgress?.(`⚠ ensureUpstreamTracking failed: ${err instanceof Error ? err.message : String(err)}`)
      }

      // ─── D-12 step 4: per-repo file ops (TRACKED) ──────────────────────────────
      for (const wsRepo of worktreeRepos) {
        if (!wsRepo.files) continue
        await runner.do(
          `apply file ops for ${wsRepo.name}`,
          async () => {
            const repoSource = { name: wsRepo.name, path: wsRepo.main_path, files: wsRepo.files }
            const result = applyFileOpsForRepo(repoSource, wsRepo)
            if (!result.ok) throw new Error(`file ops failed for ${wsRepo.name}: ${result.error}`)
            if (result.warnings) for (const w of result.warnings) onProgress?.(`file-ops: ${w}`)
          },
          async () => {
            // Strategy A (D-08): per-repo files live inside wsRepo.task_path which is
            // removed by an earlier worktree undo. The undo here is a no-op.
          },
        )
      }

      // ─── D-12 step 5: workspace-instance file ops (TRACKED) ────────────────────
      if (inputs.wsFiles) {
        await runner.do(
          `apply workspace file ops`,
          async () => {
            const result = applyFileOpsForWorkspace({ files: inputs.wsFiles }, workspaceObj, wsDir)
            if (!result.ok) throw new Error(`workspace file ops failed: ${result.error}`)
            if (result.warnings) for (const w of result.warnings) onProgress?.(`file-ops: ${w}`)
          },
          async () => {
            // Strategy A (D-08): files land inside wsDir. The wsDir itself stays so the
            // user can inspect the failure — but if it ends up empty after worktree undos,
            // remove it. Best-effort: ignore errors.
            try {
              const { readdirSync } = await import("fs")
              if (existsSync(wsDir) && readdirSync(wsDir).length === 0) {
                rmSync(wsDir, { recursive: true, force: true })
              }
            } catch { /* best-effort */ }
          },
        )
      }

      // ─── D-12 step 6: env-file writes (TRACKED) ────────────────────────────────
      if (inputs.wsEnvFile) {
        await runner.do(
          `write env files`,
          async () => {
            // mergeEnv-equivalent: workspace.env values, ports already inlined into baseEnv
            const envVars: Record<string, string> = inputs.wsEnv ? { ...inputs.wsEnv } : {}
            writeEnvFiles(workspaceObj, envVars, msg => onProgress?.(msg))
          },
          async () => {
            // Strategy A (D-08): env files land inside repo.task_path which the worktree
            // undo removes. No-op here.
          },
        )
      }

      // ─── D-12 step 7: post_create hooks (NOT tracked, D-09) ────────────────────
      if (inputs.wsHooks?.post_create?.length) {
        await runWorkspaceHooks(inputs.wsHooks.post_create, wsDir, baseEnv)
      }
    } catch (forwardError) {
      // The runner has already executed rollback by the time we land here.
      // (Plan 01's runner re-throws after running rollback so this catch block fires.)
      // Hooks throw before the runner catches them — so for hook failures we still need
      // to manually trigger the runner's rollback. The runner exposes this through
      // result() which returns the discriminated union directly.
      void forwardError
    }

    const runnerResult: RunnerResult = runner.result()
    if (!runnerResult.ok) {
      // CRITICAL: writeWorkspace is NOT called on failure (D-10 commit point).
      return { ok: false, error: runnerResult.error, rollbackErrors: runnerResult.rollbackErrors }
    }

    // ─── D-12 step 8: COMMIT POINT (D-10) — writeWorkspace OUTSIDE the runner ──
    writeWorkspace(workspaceObj)
    onProgress?.(`wrote ${inputs.wsName}.yml`)
```

### CRITICAL implementation detail — hook failures and the runner

The block above has a subtle issue: pre_create and post_create hooks run via `runWorkspaceHooks` which throws on non-zero exit. They are NOT inside `runner.do()`, so the runner does NOT catch their throws. The orchestrating try/catch above catches them, but the runner.result() at the bottom would still report `{ ok: true }` because the runner itself never saw a failure.

**Fix:** When the orchestrating try block catches a hook failure, the runner needs to know about it AND needs to roll back its accumulated stack. There are two ways:

**Approach A — manual runner trigger via a synthetic `do` call inside catch:**
```typescript
} catch (forwardError) {
  const errMsg = forwardError instanceof Error ? forwardError.message : String(forwardError)
  // Force the runner into failure state by calling do() with a forward that re-throws.
  // The runner will run rollback() and re-throw; we catch it again.
  try {
    await runner.do(
      "abort: hook failure",
      async () => { throw new Error(errMsg) },
      async () => { /* unreachable — forward throws before push */ },
    )
  } catch { /* expected — runner has now rolled back */ }
}
```

**Approach B (CLEANER — recommended): expose `runner.fail(error)` from Plan 01's runner.**

Approach B requires adding a method to the runner. **DO NOT** modify Plan 01's runner from this plan — instead use Approach A. Approach A is uglier but it preserves Plan 01's API contract and the runner's "I only know about my own do() calls" purity.
