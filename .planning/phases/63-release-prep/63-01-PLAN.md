```markdown
## [0.14.0] — 2026-04-03

### Added

**Push command** — `git-stacks push [name]` pushes all worktree repo branches to remote in parallel. Trunk repos are skipped. Supports `--force-with-lease` (safe force-push), `--force`, `--dry-run`, and `--set-upstream` flags. `--json` outputs structured `PushResult` for scripting. Per-repo progress reported inline: `pushed api (3 commits)` / `FAILED frontend: non-fast-forward`. Exit code is non-zero if any repo fails. TUI ActionMenu includes push action (`p` key) with live per-repo progress display mirroring the sync UX.

**Labels** — workspaces can be tagged with labels for organization and filtering. `git-stacks label add/remove/list/clear <workspace>` manages labels directly. `git-stacks new --label <tag>` sets labels at creation time. `git-stacks list --label <tag>` filters workspaces (AND logic with multiple `--label` flags). Labels support namespacing with colons: `sprint:14`, `client:acme`, `type:bugfix`. TUI renders label tags after the branch name, filter (`/`) matches against labels, and `g` key toggles group-by-label view with `[unlabeled]` section. Template labels are unioned onto workspaces at creation time.

**Secret references** — env values in workspace/template YAML can reference external secrets using `${{ resolver:path }}` syntax. Three built-in resolvers: `keychain` (macOS `security` / Linux `secret-tool`), `env` (process.env), and `cmd` (sh -c, opt-in only). Secrets are resolved at open time between `mergeEnv()` and `writeEnvFiles()` — resolved values are never written back to workspace YAML. `--skip-secrets` on open/new bypasses resolution and substitutes empty strings. External CLI resolvers enforce a 10-second subprocess timeout. Resolver availability is configured in `config.yml` under `secrets.resolvers`; `cmd` resolver requires explicit opt-in.

**Stash on sync** — `git-stacks sync --stash` auto-stashes dirty worktree repos before sync and pops stashes in reverse order after. Uses `git stash push --include-untracked -m "git-stacks auto-stash (sync)"` with a marker message for detection. Double-stash guard prevents stashing when a `git-stacks auto-stash` entry already exists. Stash pop conflicts set `SyncResult.ok = false`, preserve the stash entry, and emit recovery commands. TUI sync auto-stashes dirty repos without prompting.

**Ahead/behind tracking** — per-repo commit distance from base branch displayed across all surfaces. `git-stacks list` shows `↑N ↓N` indicators inline after the branch name (zeros hidden). `git-stacks status <name>` shows per-repo ahead/behind counts. `git-stacks status --fetch` triggers a fetch before computing counts. TUI WorkspaceRow displays color-coded indicators (`↑N` green, `↓N` yellow). Staleness detection flags data as stale (dim + `?` suffix) when FETCH_HEAD is older than 15 minutes. Workspace aggregate: sum of ahead across worktree repos, max behind. JSON output (`--json`) always includes numeric fields.

**Secrets config wizard** — `git-stacks config` now includes a Secrets section for toggling which secret resolvers are available. Default: `keychain` and `env` enabled, `cmd` disabled.

### Changed

**Label schema** — `WorkspaceSchema` and `TemplateSchema` now include an optional `labels` field (`string[]` with regex validation: `^[A-Za-z0-9._:-]+$`). Existing workspace YAML files without labels continue to work unchanged.

---
```
