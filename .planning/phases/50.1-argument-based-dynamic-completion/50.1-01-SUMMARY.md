## Decisions Made
- Commander.js `.command()` arg names are purely cosmetic in help output — parses by position, not name. This means zero functional change, only help text improves.
- `new [name]`, `template new [name]`, `repo add <path>`, `repo scan <dir>` intentionally NOT renamed — these are freeform or path inputs, not existing entity identifiers.
- `paths [workspace]` was already using the correct name — left unchanged.
- For `status`, `clean`, `sync` actions: introduced `const name = workspace` local alias to minimize churn in the action body.
