  <action>
    Add a new `describe("completion audit")` block at the end of the test file.

    The audit test needs to:
    1. Import and build the real Commander.js program (import the program from `src/index.ts` or build it inline by importing command registration). Since `src/index.ts` is the entry point that calls `program.parse()`, the test should either:
       - Import the program before parse is called (check if the program is exported), OR
       - Build a fresh Command and register all commands the same way index.ts does, OR
       - Use `bun run src/index.ts completion bash` via subprocess and inspect output

    The simplest approach: generate completion output using the test program extended with integration commands (from plan 01), PLUS generate output from the real program via subprocess (`Bun.spawn`). The subprocess approach tests the real program without needing to import it:

    ```typescript
    describe("completion audit - real program", () => {
      test("bash output contains integration subcommand completions", async () => {
        const proc = Bun.spawn(["bun", "run", "src/index.ts", "completion", "bash"], {
          stdout: "pipe", stderr: "pipe"
        })
        const out = await new Response(proc.stdout).text()
        expect(out).toContain("integration")
        expect(out).toContain("github")
        // Verify depth 3-4 commands appear
        expect(out).toContain("pr")
        expect(out).toContain("create")
        expect(out).toContain("issue")
        expect(out).toContain("link")
      })

      test("zsh output contains integration helper functions", async () => {
        const proc = Bun.spawn(["bun", "run", "src/index.ts", "completion", "zsh"], {
          stdout: "pipe", stderr: "pipe"
        })
        const out = await new Response(proc.stdout).text()
        expect(out).toContain("_git_stacks_integration")
        expect(out).toContain("github")
        expect(out).toContain("pr")
        expect(out).toContain("issue")
      })

      test("fish output contains multi-level integration completions", async () => {
        const proc = Bun.spawn(["bun", "run", "src/index.ts", "completion", "fish"], {
          stdout: "pipe", stderr: "pipe"
        })
        const out = await new Response(proc.stdout).text()
        expect(out).toContain("__fish_seen_subcommand_from integration")
        expect(out).toContain("github")
        expect(out).toContain("pr")
        expect(out).toContain("issue")
      })
    })
    ```

    These tests verify the REAL program's completion output, not just the test program. This catches any mismatch between what buildTestProgram() tests and what the real CLI actually generates.

    Important: Use `Bun.spawn` with `stdout: "pipe"` to capture output. The completion command writes to stdout and exits cleanly.
