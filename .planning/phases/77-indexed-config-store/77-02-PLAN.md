<objective>
Replace all raw `unlinkSync(workspacePath(...))` and `unlinkSync(templatePath(...))` calls across the codebase with the new `deleteWorkspace(name)` and `deleteTemplate(name)` functions from `config.ts`. This ensures every file deletion path evicts the corresponding cache entry (per D-02, ENGN-05).

Purpose: Without this migration, deleting a workspace/template file outside config.ts leaves a stale cache entry that causes `workspaceExists` to return true for a deleted workspace.
Output: Four files updated with cache-aware deletion calls.
</objective>
