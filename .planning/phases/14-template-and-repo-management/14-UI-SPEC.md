## Spacing Scale

Terminal UI spacing is measured in character cells (columns/rows) using OpenTUI layout props (`paddingLeft`, `paddingTop`, `height`). All declared values are multiples of 1-cell increments; the minimum addressable unit is 1 cell.

| Token | Character Cells | OpenTUI Prop Usage | Notes |
|-------|----------------|--------------------|-------|
| xs | 1 | `paddingLeft={1}` | Icon-glyph gap, inline separator |
| sm | 2 | `paddingLeft={2}` | Action menu item indent, detail box indent |
| md | 4 | `paddingLeft={4}` | Nested sub-item indent (e.g. conflict file list) |
| row | 1 | `height={1}` | Standard list row height |
| section-gap | 1 | blank `<text>{""}` | Separator between sections in detail pane |
| modal-top | 1‚Äì2 | `paddingTop={1}` or `paddingTop={2}` | Action menu and wizard top padding |

Exceptions:
- `paddingTop={2}` is used in ConfirmDialog for visual breathing room (source: ConfirmDialog.tsx line 17).
- Conflict sub-rows use 5-space indent (5 literal spaces) to align under the spinner glyph column (source: SyncProgressView.tsx line 62).

Source: codebase ‚Äî existing components WorkspaceRow, ConfirmDialog, SyncProgressView, TemplateActionMenu.

## Typography

Terminal UI has no font-size or weight control. "Typography" here means column widths and foreground colors that signal information hierarchy.

| Role | Column Width | Color | Usage |
|------|-------------|-------|-------|
| Primary label | variable | `fg="white"` | Repo name, workspace name, template name, heading text |
| Secondary / metadata | variable | `fg="gray"` | Path, type, description, key-binding hints, timestamps |
| Active/interactive | variable | `fg="cyan"` | Branch name, repo count, repo type column, active spinner |
| Success / disk-exists indicator | 1 glyph | `fg="green"` | ‚úì glyph for disk-exists, sync success row |
| Error / missing indicator | 1 glyph | `fg="red"` | ‚úó glyph for missing, sync failure row, validation error text |
| Warning / skipped indicator | 1 glyph | `fg="yellow"` | ‚öÝ glyph for skipped, confirm dialog message, wizard confirm message |
| Focused row highlight | full row | `backgroundColor="#333333"` | Cursor row across all lists |

Source: codebase ‚Äî WorkspaceRow, RepoList, TemplateList, SyncProgressView, WizardView, ConfirmDialog.

## Color

Terminal UI "color" is OpenTUI named foreground/background values, not hex. The 60/30/10 model maps to:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) ‚Äî surfaces | (terminal default bg) | All list rows not focused, detail pane background |
| Secondary (30%) ‚Äî highlight | `backgroundColor="#333333"` | Focused/cursor row in WorkspaceList, RepoList, TemplateList |
| Accent (10%) ‚Äî interactive data | `fg="cyan"` | Branch name column, repo count column, repo type column, active progress spinner |
| Success semantic | `fg="green"` | Disk-exists glyph (‚úì), sync-synced row status, summary "N synced" text |
| Warning semantic | `fg="yellow"` | Confirm dialog message, wizard confirm step message, sync-skipped row status |
| Destructive semantic | `fg="red"` | Disk-missing glyph (‚úó), validation errors, sync-failed row status, remove-blocked dialog |

Accent (`fg="cyan"`) reserved for:
- Branch name column in workspace rows
- Repo count column in template rows
- Repo type column in repo rows
- Active-state spinner (fetching/rebasing in SyncProgressView)
- NOT used for key-binding hints or static labels

## Interaction Contract

### RepoActionMenu keyboard bindings

```
[w]   Create workspace (from selected repos, or focused repo if none selected)
[t]   Create template  (from selected repos, or focused repo if none selected)
[r]   Remove           (focused repo, or batch if multiple selected)
[Esc] Back             (return to list view)
```

Label format when multi-select is active (N repos selected):
- `[w] Create workspace (N repos)`
- `[t] Create template (N repos)`
- `[r] Remove (N)`

Label format when single repo (no multi-select):
- `[w] Create workspace`
- `[t] Create template`
- `[r] Remove`

Source: 14-CONTEXT.md D-04, D-05.

### Template create wizard flow

Steps in order:
1. Text step ‚Äî label: `Template name:` ‚Äî validates uniqueness via `templateExists()` ‚Äî error color: `fg="red"`
2. Confirm step ‚Äî message format: `Create template "{name}" with {N} repos?` ‚Äî keys: `[y] Create  [Esc] Back`

After creation: switch to Templates tab, cursor positioned on new template row.

Source: 14-CONTEXT.md D-16, D-17, D-18.

### Repo remove flow

**Path A ‚Äî references exist (blocked):**
Show RemoveBlockedView (not ConfirmDialog):
- Header: `Cannot remove "{name}" ‚Äî referenced by:` (fg="yellow")
- List each reference: `  template: {name}` or `  workspace: {name}` (fg="gray")
- Footer: `Remove these references first, then retry.` (fg="gray")
- Key: `[Esc] Back`

**Path B ‚Äî no references (safe remove):**
Show ConfirmDialog with message: `Remove repo "{name}" ({path})?`
- Keys: `[y] Yes  [n/Esc] No`
- After confirm: reload repos list, cursor clamped to valid range.

Source: 14-CONTEXT.md D-07, D-08, D-09; RepoDetail.tsx reference logic.

### Keyboard isolation guards

All new modal views (`RepoActionMenu`, `RemoveBlockedView`, wizard) must follow the established guard pattern in App.tsx: place `if (v.view === "repo-action-menu") return` (and equivalents) above all list navigation handlers in the main `useKeyboard` callback.

Deferred focus pattern applies to all WizardView instantiations: mount `focused={false}`, `setTimeout(() => setFocused(true), 0)`.

Source: CLAUDE.md keyboard isolation rules; 14-CONTEXT.md established patterns.
