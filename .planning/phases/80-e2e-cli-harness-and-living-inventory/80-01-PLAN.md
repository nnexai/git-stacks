<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| test file → spawned CLI subprocess | Untrusted argv/env/cwd cross into a real process |
| harness → filesystem | Test helpers create and mutate temp config, repo, and YAML files |
| failure bundle → terminal output | Debug output can disclose sensitive env data if not reduced |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-80-01 | T | `tests/helpers.ts` `runCli` | mitigate | Force isolated `HOME`, `XDG_CONFIG_HOME`, `GNUPGHOME`, `GIT_CONFIG_GLOBAL`, `GIT_CONFIG_NOSYSTEM`, `GIT_TERMINAL_PROMPT=0`, and `GIT_STACKS_CONFIG_DIR` from `getTestGitEnv(opts.baseDir)` before every spawn. |
| T-80-02 | I | `tests/helpers.ts` `formatCliFailure` | mitigate | Emit only `TEST_CLI_ENV_ALLOWLIST` keys plus explicit `envAllowlistExtras`; never stringify `process.env` wholesale. |
| T-80-03 | T | harness cwd/path handling | mitigate | Capture explicit `cwd`, accept explicit `artifactPaths`, and prove in tests that persisted files are under the temp fixture root. |
| T-80-04 | T/E | subprocess launch assembly | mitigate | Use `Bun.spawnSync(["bun","run","src/index.ts", ...argv])` argv arrays only; forbid `sh -c` and string-built shell commands inside the harness. |

No high-severity threat remains after these mitigations.
</threat_model>
