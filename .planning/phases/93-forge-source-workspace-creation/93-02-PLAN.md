# Phase 93 Plan 02 - Source Fetch And Worktree Handoff

<objective>
Implement the plain Git source fetch and worktree branch handoff so the matched template repo starts from the forge source ref while every other template repo follows normal workspace branch creation.
</objective>

<tasks>
  <task id="93-02-01" type="auto">
    <name>Add plain Git source fetch helpers and source branch conflict checks</name>
    <read_first>
      - src/lib/git.ts
      - tests/lib/git-real-remote.test.ts
      - tests/commands/workspace-git-ops.test.ts
      - .planning/phases/92-forge-source-research-and-resolver-design/92-04-PLAN.md
    </read_first>
    <files>src/lib/git.ts, src/lib/workspace-source.ts, tests/lib/workspace-source.test.ts</files>
    <action>In `src/lib/git.ts`, add helpers for source creation: `fetchSourceRef(repoPath, remote, sourceRef, fetchedRef)` using `GIT_TERMINAL_PROMPT=0 git -C <repoPath> -c fetch.timeout=30 fetch <remote> <sourceRef>:<fetchedRef>`, `deleteRef(repoPath, ref)` using `git update-ref -d`, and `resolveRef(repoPath, ref)` returning the resolved SHA or a typed failure. In `src/lib/workspace-source.ts`, call these helpers for the matched worktree repo before workspace creation. Derive `fetchedRef` as `refs/git-stacks/sources/<forge>/<change-number>/<repo-name>`. Before using a source branch, check an existing local branch of the same name: if branch metadata or resolved SHA does not match the fetched source ref, fail with `branch_conflict`; do not reset or overwrite the branch. Fetch failures must format provider-specific guidance naming the forge, URL, source ref, and auth/fork access checks.</action>
    <verify>
      <automated>bun test tests/lib/workspace-source.test.ts</automated>
    </verify>
    <acceptance_criteria>
      - `src/lib/git.ts` contains `fetchSourceRef`.
      - `src/lib/git.ts` contains `deleteRef`.
      - `src/lib/workspace-source.ts` uses `refs/git-stacks/sources/`.
      - A test asserts fetch failure output mentions auth or credentials and fork access.
      - A test asserts branch conflict does not call any reset operation.
    </acceptance_criteria>
    <done>Matched source refs are fetched through plain Git into operation-owned internal refs, with clear conflict and fetch failure behavior.</done>
  </task>

The narrow implementation may add `sourceStartRefs?: Record<string, string>` to `CreateWorkspaceInputs` and use a new `createWorktreeFromRef(repo.main_path, repo.task_path, branch, fetchedRef)` only for the matched repo. Keep `writeWorkspace()` as the commit point. Add `tests/commands/workspace-source-git.test.ts` using `makeRepoWithRemote()`: create `api` and `web` repos in a template, push `feature/mr-42` to the api bare remote, run `git-stacks new review-42 --non-interactive --template review --source https://gitlab.example.com/org/api/-/merge_requests/42`, then assert api task worktree HEAD equals the source branch commit and web task worktree is on the workspace branch created by normal logic.</action>

Add a unit test that mocks `createWorkspace()` returning `ok: false`, verifies `deleteRef()` is called exactly for the fetched internal ref, and verifies no workspace YAML write is performed.</action>

    <done>Fetch side effects owned by source creation are rolled back if later workspace creation fails.</done>
  </task>
