<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workspace YAML -> shell command execution | User-authored manual commands execute shell through existing command machinery. |
| command output -> dashboard progress | Failures need enough context for the operator to inspect before returning to the list. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-06 | Elevation of Privilege | manual command execution | accept | Manual commands are explicitly user-authored operator actions from Phase 95; Phase 99 only exposes the existing manual trigger in the TUI. |
| T-99-07 | Information Disclosure | hidden command names | mitigate | Use `listManualCommands()` default filtering so `pre*` and `post*` buckets are not shown as primary picker entries. |
| T-99-08 | Repudiation | failed command output | mitigate | Include failed command context in `ProgressView` and keep the view until keypress. |
| T-99-SC | Tampering | dependencies | mitigate | No dependency changes; use existing command execution helpers. |
</threat_model>
