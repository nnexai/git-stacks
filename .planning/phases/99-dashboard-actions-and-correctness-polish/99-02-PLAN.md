<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workspace YAML -> issue opener | User-authored integration issue metadata selects a tracker and issue id to open. |
| external tracker CLI/browser -> dashboard | Issue open failures must be visible to the operator instead of hidden by a return to list. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-03 | Spoofing | linked issue labels | mitigate | Display tracker name and issue id from workspace settings and route only known tracker ids. |
| T-99-04 | Repudiation | issue-open failures | mitigate | Keep progress/error output visible until keypress so the selected tracker and failure text remain inspectable. |
| T-99-05 | Denial of Service | disabled issue row | mitigate | Disabled rows are visible but non-activating for both Enter and shortcut dispatch. |
| T-99-SC | Tampering | dependencies | mitigate | No dependency changes; use existing issue integration helpers or CLI command surfaces. |
</threat_model>
