<threat_model>
## STRIDE threat register

| Category | Threat | Disposition and mitigation |
|---|---|---|
| Spoofing | Workspace input forges service-owned `GS_*` identity. | Mitigate by writing reserved keys last and testing conflicting overlays. |
| Tampering | User command text becomes executable bootstrap source. | Mitigate with fixed bootstrap code and one unchanged positional command argument. |
| Repudiation | Shell startup failure is indistinguishable from command failure. | Mitigate with structured stage/class diagnostics. |
| Information Disclosure | Secrets or environment values leak into diagnostics. | Mitigate with metadata-only diagnostics and negative disclosure tests. |
| Denial of Service | Initialization or descendants outlive cancellation. | Mitigate with initialization-only deadline and process-group TERM/grace/KILL. |
| Elevation of Privilege | Unsupported or attacker-controlled fallback shell executes input. | Mitigate with absolute executable validation, allowlisted families, and no fallback. |
</threat_model>
