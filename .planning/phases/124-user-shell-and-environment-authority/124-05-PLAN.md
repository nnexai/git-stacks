## STRIDE threat register

| Category | Threat | Disposition and mitigation |
|---|---|---|
| Spoofing | Browser/remote surface masquerades as trusted refresh submitter. | Construct refresh only in authenticated local CLI/TUI code and retain router denial tests. |
| Tampering | Stale environment merges with omitted replacement keys. | Submit complete replacement and assert clear-on-omission end to end. |
| Repudiation | Hosted capability is skipped while CI appears green. | Required env flags convert absence/skip into hard failure. |
| Information Disclosure | Shell/SSH fixtures print socket, PATH, keys, or secrets. | Use canaries, metadata-only diagnostics, redacted captures, and negative scans. |
| Denial of Service | Agents or process trees survive failed tests/cancellation. | Structured finally cleanup plus production TERM/grace/KILL assertions. |
| Elevation of Privilege | Refresh points launches at an untrusted remote agent. | Same-user local authorization, allowlisted keys, and no remote delegation. |
</threat_model>
