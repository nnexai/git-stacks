  <action>
The root cause of tab switching freeze (UAT tests 2, 3) and help overlay sticking is that OpenTUI's terminal renderer doesn't repaint when SolidJS conditional rendering (Show, Switch/Match) restructures the DOM tree. The signal/state layer works (events dispatch, state preserved), but the terminal buffer keeps the previous content painted.

**Fix: render everything permanently, toggle visibility via height.**

Read `src/tui/dashboard/App.tsx` first.

**Step 1: Replace help overlay Show blocks (lines ~516-653) with height-based toggle.**

Replace the two `<Show>` blocks wrapping the help overlay and main content:

```tsx
{/* OLD — two Show blocks that swap */}
<Show when={helpOpen()}>
  <HelpOverlay ... />
</Show>
<Show when={!helpOpen()}>
  {/* main content */}
</Show>
```

With height-based visibility — both always rendered, height toggled:

```tsx
{/* Help overlay — full height when open, hidden when closed */}
<box height={helpOpen() ? "100%" : 0} overflow="hidden">
  <HelpOverlay tab={tab()} onClose={() => setHelpOpen(false)} />
</box>

{/* Main content — full height when help closed, hidden when open */}
<box flexDirection="column" height={helpOpen() ? 0 : "100%"} overflow="hidden">
  {/* TOP BOX and BOTTOM BOX and HELP BAR go here — unchanged internally except Step 2 */}
</box>
```

Note: Keep `flexDirection="column"` on the main content wrapper. The `overflow="hidden"` ensures hidden content doesn't bleed.

<task type="auto">
  <name>Task 2: Fix rename remnants — reset view to list after rename completes</name>
  <files>src/tui/dashboard/App.tsx</files>
  <action>
The root cause of rename remnants (UAT test 8) is in `handleInlineInputConfirm()`. After a rename completes, the function calls `setProgressDone(true)` then `reload()` but never calls `setView({ view: "list" })`. The view stays on 'progress' until the user presses a key.

Read `src/tui/dashboard/App.tsx` first.

**Fix:** In the rename branch of `handleInlineInputConfirm()`, change lines 310-313 from:

```typescript
if (!result.ok) setProgressLines(prev => [...prev, `ERROR: ${result.error}`])
setProgressDone(true)
reload()
return
```

To:

```typescript
if (!result.ok) {
  setProgressLines(prev => [...prev, `ERROR: ${result.error}`])
  setProgressDone(true)
  // Stay on progress view so user sees the error
} else {
  setProgressDone(true)
  reload()
  setView({ view: "list" })
}
return
```

This preserves error visibility (user sees error on progress view) while fixing the clean-completion path (view resets to list immediately after successful rename).
