<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args / cwd / env -> workspace resolution | Untrusted invocation context crosses into choosing which workspace note file can be read or deleted. |
| Destructive clear command -> filesystem delete | Operator intent crosses into irreversible local note-file deletion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-96-04 | Spoofing | `src/commands/notes.ts` workspace resolver | mitigate | Enforce the D-03 precedence order explicitly and require the resolved workspace to exist before reading or deleting any note file. |
| T-96-05 | Tampering | `src/commands/notes.ts` clear path | mitigate | Require confirmation unless `--force` is present per D-06, and delete only the resolved per-workspace note file returned by the storage helper. |
| T-96-06 | Tampering | `src/commands/notes.ts` malformed-store handling | mitigate | Propagate the Plan 96-01 corruption error without mutating the file so a bad store cannot be silently cleared or appended to. |
</threat_model>
