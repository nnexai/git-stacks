<feature>
  <name>Workspace CWD Detection Core</name>
  <files>src/lib/workspace-ops.ts, src/lib/integrations/issue-utils.ts, tests/lib/detect-workspace-cwd.test.ts</files>
  <behavior>
    detectWorkspaceFromCwd:
    - Given CWD exactly equals a worktree task_path -> returns { ok: true, workspace } with that workspace
    - Given CWD is a subdirectory of a worktree task_path (e.g., task_path + "/src/components") -> returns { ok: true, workspace }
    - Given CWD does not match any worktree task_path -> returns { ok: false, error: "no_match" }
    - Given CWD matches task_paths in two different workspaces, returns the workspace with the longest (deepest) task_path match
    - Given a workspace has trunk-mode repos, those repos are never considered for matching (only worktree-mode repos)
    - Given task_path contains ~/..., the path is normalized via resolve(expandHome()) before comparison
    - Given optional cwd parameter, uses that instead of process.cwd() (for testability)
    - Given a task_path like "/tmp/tasks/my-ws/repo-na", CWD "/tmp/tasks/my-ws/repo-name" does NOT match (trailing separator guard)

    resolveWorkspaceArg:
    - Given explicit workspaceName string, validates it exists and returns it
    - Given explicit workspaceName that does not exist, calls process.exit(1) with error message
    - Given undefined workspaceName, calls detectWorkspaceFromCwd() and returns detected workspace name
    - Given undefined workspaceName and no CWD match, calls process.exit(1) with descriptive error including tracker and action names
  </behavior>

    2. Add resolveWorkspaceArg() to issue-utils.ts
    3. Update formatIssueError for no_issue_linked to mention CWD detection as alternative usage
  </implementation>
</feature>
