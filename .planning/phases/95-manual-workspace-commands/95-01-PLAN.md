<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Template/workspace YAML -> saved workspace config | User-authored command strings cross into persisted workspace state and later become executable shell. |
| Included templates -> composed template result | Colliding command names from multiple templates can silently change what will later execute. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-01 | Tampering | `src/lib/config.ts` command schema | mitigate | Keep the schema to string-valued maps only per D-01 through D-03; reject object-valued metadata that would widen the execution surface. |
| T-95-02 | Tampering | `src/lib/composition.ts` template command merge | mitigate | Encode last-write-wins collisions in tests and implementation per D-08 so command overrides are deterministic and documented. |
| T-95-03 | Repudiation | `src/lib/workspace-lifecycle.ts` persistence contract | mitigate | Persist copied commands directly into workspace YAML so later execution does not depend on mutable template state. |
</threat_model>
