<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Saved workspace YAML -> shell execution | User-authored command strings become directly executed shell commands. |
| Secret resolution -> streamed terminal output | Resolved env can influence child process behavior and may be exposed by the child command itself. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-07 | Elevation of Privilege | `src/lib/workspace-command.ts` execution engine | accept | The feature explicitly executes user-authored shell; keep it manually triggered, narrow in schema, and inspectable via dry-run rather than auto-running or widening metadata. |
| T-95-08 | Information Disclosure | `buildWorkspaceEnv()` + terminal streaming | mitigate | Support `skipSecrets` per D-24 and keep Phase 95 human-first with direct output only, no JSON artifact persistence. |
| T-95-09 | Tampering | Command ordering and scope resolution | mitigate | Encode D-05 and D-10 ordering in unit tests so workspace/repo/pre/post semantics stay deterministic. |
| T-95-10 | Repudiation | Failure propagation | mitigate | Preserve the exact failing exit status from the streamed shell helper so callers and users can identify the real failing command outcome. |
</threat_model>
