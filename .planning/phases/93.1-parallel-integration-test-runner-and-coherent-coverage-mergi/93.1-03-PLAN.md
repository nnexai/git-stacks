<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Coverage worker env -> shard directories | Worker-specific environment variables control where coverage JSON is written. |
| Coverage merge step -> release-gate artifacts | Partial or malformed worker output must not corrupt canonical `.coverage/*` artifacts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-93-1-07 | Tampering | Coverage shard directories | mitigate | Use per-file/process coverage directories and merge only after all workers finish per D-14. |
| T-93-1-08 | Repudiation | Partial-failure coverage runs | mitigate | Merge successful shards even when one worker fails, then exit nonzero with canonical artifacts for debugging per D-15. |
| T-93-1-09 | Availability | Coverage debug evidence | mitigate | Preserve worker directories automatically on failure and on `--keep-workdir`; only clean them after successful merges per D-17. |
</threat_model>
