## Phase Boundary

Phase 93.1 makes isolated integration and coverage runs faster through bounded parallel execution while preserving the existing coherent file-level pass/fail summary and merged Istanbul coverage artifacts. The phase is local-gate focused; it does not introduce CI requirements.

</domain>

<decisions>
## Implementation Decisions

### Parallelism Defaults
- **D-01:** Default integration worker count is fixed and conservative at 4 workers.
- **D-02:** Worker count is overridden by CLI flag only: `--workers <n>`.
- **D-03:** `--workers 1` is the official serial compatibility/debug mode and must preserve current serial semantics.
- **D-04:** Worker count applies to both integration test mode and coverage integration mode.

### Output Semantics
- **D-05:** Buffer stdout/stderr per test file and print a file's output after that file completes to avoid interleaved logs.
- **D-06:** Print completed file results in completion order so users see progress as workers finish.
- **D-07:** End with a file-attributed failure summary: failed file, exit code, and a short tail/excerpt of buffered output.
- **D-08:** Preserve today's file-level final count semantics, such as `N/M integration files passed`.

### Fixture Isolation
- **D-09:** Each integration file still runs in its own process and inherits its existing per-file temp/config setup.
- **D-10:** Phase 93.1 should make every current integration file parallel-safe where feasible.
- **D-11:** Add a serial bucket only if a specific file proves impractical after investigation, and document the reason.
- **D-12:** Add targeted tests/guards for temp directory, config directory, local Git fixture, and fixed-path collision regressions where the runner can catch them.
- **D-13:** Do not fail fast by default. Run the full scheduled suite and report all failures.

### Coverage Merge Behavior
- **D-14:** Coverage integration runs use per-file/process coverage directories and merge after all workers finish.
- **D-15:** If one coverage worker fails, still merge artifacts from successful files for debugging, then exit nonzero and mark the coverage run failed.
- **D-16:** Preserve canonical final output paths: `.coverage/coverage-final.json`, `.coverage/coverage-summary.json`, `.coverage/lcov.info`, and `.coverage/index.html`.
- **D-17:** Clean worker temp coverage directories after a successful merge. Preserve them on failure or with a debug flag.

<specifics>
## Specific Ideas

- A fixed default of 4 workers is preferred over CPU-based defaults to keep local behavior predictable.
- A serial bucket is acceptable only as a documented exception after attempting to make current files parallel-safe.
- Coverage merge should be useful for debugging even on failed runs by merging successful worker artifacts before returning nonzero.

</specifics>
