### Bash generator changes

**In `generateBash()`, add a prev-word detection block AFTER the `prev="${COMP_WORDS[COMP_CWORD-1]}"` line and BEFORE the `case "${words[1]}" in` line.**

The generated bash script must include a section like:

```bash
  # Fixed-choice flag values
  case "$prev" in
    --strategy)
      COMPREPLY=($(compgen -W "rebase merge" -- "$cur"))
      return 0
      ;;
    --sort)
      COMPREPLY=($(compgen -W "date name status" -- "$cur"))
      return 0
      ;;
  esac

  # Dynamic flag-value completions
  case "$prev" in
    --workspace)
      local names
      names=$(ls "$HOME/.config/${name}/workspaces" 2>/dev/null | sed 's/\.yml$//')
      COMPREPLY=($(compgen -W "$names" -- "$cur"))
      return 0
      ;;
  esac
```

Implementation approach: In the `generateBash()` function, after the `prev=...` line and before the `case "${words[1]}"` line, iterate over `OPTION_ENUMS` entries to emit a case statement, then iterate over `FLAG_COMPLETIONS` entries to emit a second case statement (or combine into one case). Use `bashDynamicLookup()` for FLAG_COMPLETIONS values.

Concretely, add this code generation block in `generateBash()` after line `out += '  words=("${COMP_WORDS[@]}")\n'` and before the `if [[ ${COMP_CWORD} -eq 1 ]]` block:

```typescript
  // Emit OPTION_ENUMS prev-word detection
  const enumEntries = Object.entries(OPTION_ENUMS)
  const flagEntries = Object.entries(FLAG_COMPLETIONS)
  if (enumEntries.length > 0 || flagEntries.length > 0) {
    out += "\n"
    out += '  case "$prev" in\n'
    for (const [flag, values] of enumEntries) {
      out += `    ${flag})\n`
      out += `      COMPREPLY=($(compgen -W "${values.join(" ")}" -- "$cur"))\n`
      out += `      return 0\n`
      out += `      ;;\n`
    }
    for (const [flag, dynType] of flagEntries) {
      out += `    ${flag})\n`
      out += bashDynamicLookup(dynType, "      ", name)
      out += `      return 0\n`
      out += `      ;;\n`
    }
    out += "  esac\n"
  }
```

```typescript
      if (sub.options.length === 0) continue
      out += `\n# Flags for ${node.name} ${sub.name}\n`
      for (const opt of sub.options) {
        const longName = opt.long.slice(2)
        out += `complete -c ${name} -f -n '__fish_seen_subcommand_from ${node.name}; and __fish_seen_subcommand_from ${sub.name}' -l ${longName} -d '${opt.description}'\n`
      }
    }
```

Note: The OPTION_ENUMS and FLAG_COMPLETIONS sections use `-ra` (requires argument + argument list) which overrides the basic flag directive from this section. Fish uses the most specific match, so the `-ra` directives take precedence when the flag expects a value.
