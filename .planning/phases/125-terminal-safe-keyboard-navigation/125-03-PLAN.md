<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser keyboard event to app matcher | Untrusted layout/composition/modifier state must not be confused with app intent. |
| Projected signals to navigation target | Stale or inconsistent projection data must not redirect to inaccessible state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-125-04 | Spoofing | shortcut matcher | high | mitigate | Match exact normalized physical code/modifiers and reject composition/AltGraph/extra modifiers before action resolution. |
| T-125-05 | Denial of service | fuzzy ranking | medium | mitigate | Operate on already bounded projections with deterministic linear scoring and stable tie-breakers; no regex from user query. |
| T-125-06 | Tampering | attention candidate input | medium | mitigate | Require active workspace/repository membership and exact live-surface resolution before selection. |
| T-125-SC | Tampering | dependency supply chain | low | accept | No new fuzzy, keyboard, or UI package is installed; algorithms use existing client code and language primitives. |
</threat_model>
