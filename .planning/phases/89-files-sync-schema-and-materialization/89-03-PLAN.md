<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sync config -> local git exclude | User-controlled target paths influence local ignore rules. |
| Worktree -> common git dir | Linked worktree metadata resolves where ignore rules are written. |
| File materialization -> exclude mutation | A filesystem write must complete before ignore rules are updated. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-04 | Tampering | Local exclude file | mitigate | Write only to `<common-git-dir>/info/exclude`; never modify project `.gitignore`. |
| T-89-05 | Tampering | Exclude path construction | mitigate | Build rooted exclude patterns from validated relative sync targets only. |
| T-89-06 | Repudiation | Existing user excludes | mitigate | Preserve existing lines and dedupe appended sync entries. |
| T-89-07 | Denial of Service | Non-git repo with `git_exclude: true` | mitigate | Return a clear failure instead of silently materializing unexcluded files. |
</threat_model>
