<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service snapshot/event → reducer | Potentially malformed, stale, duplicate, incompatible, or unknown contract data influences product state/effects. |
| Session file → restored model | Local filesystem bytes may be corrupt, symlinked, permission-unsafe, or crafted to claim false liveness. |
| Reducer effects → platform adapters | Pure model decisions become privileged service, terminal, persistence, or focus operations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-105-04 | Tampering | reducer action admission | high | mitigate | Strict schemas, ordered revision/sequence checks, inert unknown optionals, and golden transition tests. |
| T-105-05 | Information Disclosure | session persistence/diagnostics | high | mitigate | Presentation allowlist, owner-only atomic storage, bounded redacted diagnostics, and negative byte scans. |
| T-105-06 | Spoofing | restored process liveness | high | mitigate | Restore every entry as ended and create a new identity for relaunch; never persist or probe OS identifiers. |
| T-105-07 | Elevation of Privilege | effect emission | high | mitigate | Tagged effects from validated transitions only; incompatible/unknown data cannot emit effects. |
</threat_model>
