<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terminal child/process group → native client | Untrusted subprocesses, descendants, escape content, and exit timing cross an ownership boundary. |
| Native client → ownership guard | Privileged PID/PGID registration and teardown commands cross a private IPC boundary. |
| GTK callbacks/render/input → libghostty adapter | Asynchronous native callbacks can race surface/GPU destruction and carry clipboard/input data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-105-08 | Elevation of Privilege | PGID signaling | critical | mitigate | Validate session/group/birth token, reject client/guard groups, use direct POSIX APIs, and test reuse/rapid-close races. |
| T-105-09 | Denial of Service | terminal teardown | high | mitigate | Bounded injected escalation/reaping and visible failed_cleanup instead of hanging or discarding ownership. |
| T-105-10 | Tampering | guard command channel | high | mitigate | Private inherited channel, strict framed commands, no public PID endpoint, registration lifecycle tests. |
| T-105-11 | Information Disclosure | diagnostics/clipboard | high | mitigate | Treat terminal/clipboard bytes as data and redact argv, env, credentials, clipboard, and unrestricted output from diagnostics. |
| T-105-12 | Tampering | late GTK/libghostty callback | high | mitigate | Generation/liveness token, controller disconnection, join/stop ordering, and automated post-unrealize rejection tests. |
</threat_model>
