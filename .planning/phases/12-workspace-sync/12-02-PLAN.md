Helper functions:

```typescript
function glyphFor(status: SyncRow["status"]): string {
  if (status === "pending")  return "·"
  if (status === "synced")   return "✓"
  if (status === "skipped")  return "⚠"
  if (status === "failed")   return "✗"
  return ""  // fetching/rebasing use spinner
}

function colorFor(status: SyncRow["status"]): string {
  if (status === "pending")  return "gray"
  if (status === "synced")   return "green"
  if (status === "skipped")  return "yellow"
  if (status === "failed")   return "red"
  return "cyan"  // fetching/rebasing
}
```

Props type — summary is an object with `text` and `color`:

```typescript
type Props = {
  rows: SyncRow[]
  done: boolean
  summary: { text: string; color: "green" | "yellow" | "red" }
}

export function SyncProgressView(props: Props) { ... }
```

Render structure per UI-SPEC:
- `<Show when={!props.done}>` header with spinner + "Syncing..." text
- `<For each={props.rows}>` per-row rendering:
  - `<box flexDirection="row" height={1} paddingLeft={2}>`
  - For fetching/rebasing: `<spinner name="dots" color="cyan" />` + space
  - For other statuses: `<text fg={colorFor(status)}>{glyphFor(status)} </text>`
  - `<text fg="white">{row.repo}</text>`
  - `<text fg="gray">  {row.detail}</text>`
  - `<For each={row.conflicts}>` conflict files indented 5 cols, fg="gray"
- `<Show when={props.done && props.summary.text}>` summary line with `fg={props.summary.color}`

Use `<>...</>` (fragment) to wrap each row + its conflict lines inside the For callback — same as the pattern shown in 12-RESEARCH.md Pattern 7.
