<objective>
Add dir-mode guards to all workspace lifecycle functions in workspace-ops.ts so that workspaces containing dir repos can be created, opened, closed, cleaned, removed, and renamed without TypeScript errors or runtime crashes.

Purpose: Phase 64 made task_path optional and added mode "dir" to schemas. Every function in workspace-ops.ts that dereferences task_path or runs git operations on repos must now guard against dir repos. Without these guards, the codebase has TypeScript compilation errors and runtime crashes for any workspace containing a dir repo.

Output: A fully type-safe workspace-ops.ts that handles the three-mode discriminant (worktree/trunk/dir), plus tests proving dir repos flow through all lifecycle operations correctly.
</objective>
