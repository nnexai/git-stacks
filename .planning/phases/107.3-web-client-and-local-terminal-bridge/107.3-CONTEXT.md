# Phase 107.3 Context

## Locked Decisions

- The Bun/TypeScript service is the only browser-facing machine authority.
- The browser renders browser-safe workspace state and xterm.js terminals; it never reads configuration, runs Git, constructs launch commands, or receives service credentials and raw launch environments.
- Web terminal creation accepts stable workspace, repository, optional command, revision, and dimension inputs only.
- The service owns browser PTYs, process groups, output retention, signal filtering, pressure limits, and cleanup.
- Access is same-origin and loopback-only through one-use pairing and a scoped HttpOnly principal.
- Reconnect uses cursor-based replay. Replay gaps and service restarts are explicit rather than silently partial.
- Terminal activity signals are exact-surface scoped and are removed when their service-owned terminal tab is closed.
- External terminal integrations remain separate session owners.

## Deferred

- Service-restart and machine-reboot terminal durability.
- Shared writers or multi-user collaboration.
- LAN, public, or hosted access.
- Platform terminal capability without actual-host lifecycle verification.
