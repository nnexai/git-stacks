# Phase 107.3 Implementation Summary

**Completed:** 2026-07-13
**Verified runtime:** Linux, Bun 1.3.14, Chromium through `playwright-cli`
**Result:** Production service-owned web client implemented; native interaction parity delivered within the browser-relevant scope.

## Outcome

`git-stacks web` now starts or discovers the existing managed service, issues a one-use loopback pairing URL, exchanges the fragment for an HttpOnly browser principal, and serves a thin local browser client from the same service listener. The official service bearer never enters the browser.

All machine authority stays in TypeScript service code: workspace/Git observation, launch resolution, PTY ownership, process cleanup, cursor replay, quotas, operations, workspace creation, event projection, and terminal signal ingestion. Browser code only renders browser-safe DTOs and sends stable IDs plus user intents.

## Implemented service architecture

- One `Bun.serve` listener keeps the existing authenticated `/v1` contract and adds an isolated `/web` policy/serialization boundary.
- One-use 60-second pairing codes are stored hashed and exchanged for opaque Strict HttpOnly principals with idle/hard expiry and revocation.
- Browser snapshots, catalogs, signals, and operations use explicit allowlist projections. Paths, launch commands, cwd, environment, references, ports, credentials, PIDs, and terminal bytes never enter generic browser DTOs or diagnostics.
- Browser PTYs are launched only through the authoritative native launch resolver using workspace, repository, command, and revision IDs.
- Each web session owns its PTY/process group, bounded output ring, authenticated OSC signal filter, one writable WebSocket attachment, ACK cursor, resize/input controls, and deterministic TERM/KILL cleanup.
- SSE reuses the shared event broker and filters browser operations by principal ownership. Existing official/native events and credentials remain isolated.
- Explicit terminal close removes the service session permanently. A naturally exited interactive shell removes its tab, while configured-command sessions retain their ended output without a relaunch action.

## Browser feature parity

- Workspace-first navigation with shallow single-repository rows and explicit multi-repository children.
- Persistent exact workspace/repository selection, dedicated pinned and active sections, workspace-definition pin membership, general numeric priority inside every group, and label/repository organization modes.
- Git dirty/ahead/behind/degraded state plus workspace, repository, and direct terminal attention counts.
- Independent terminal collections per exact pair; hidden tabs retain their live process and xterm state.
- Multiple tabs, keyboard switching, drag reorder, F2 rename, selection copy, transactional close, interactive-shell removal on exit, and retained configured-command output.
- Real resize propagation, reconnect with takeover/backoff, cursor de-duplication, reload restoration, and explicit retained-history loss messaging.
- Configured-command-only searchable launcher with recent-first order and exact scope labels; launch failure leaves the launcher available.
- Workspace creation from templates or selected repositories and allowlisted workspace open/close operations through the shared registry.
- Signal inbox, explicit focus/fallback, dismissal, no asynchronous focus theft, theme cycling, responsive layout, reduced-motion handling, live regions, and keyboard-operable dialogs.

## Packaging and operation

The deterministic browser bundle is emitted to `dist/web` and included in the npm package. Source checkouts build it on demand; publish runs build it before packaging.

```bash
git-stacks web
git-stacks web --no-open
git-stacks web --no-open --json
```

The service remains loopback-only. Web terminals are separate from native Ghostty and external-integration sessions, have one browser writer, are process-local, and do not survive a managed-service restart.

## Honest capability boundary

Linux is enabled and verified. macOS and Windows discovery reports browser terminals unavailable because actual-host PTY/process-tree cleanup evidence has not been earned. Chrome/Chromium was exercised end to end; Firefox, Safari, Edge, IME, and screen-reader product matrices remain release-environment validation rather than claimed evidence.
