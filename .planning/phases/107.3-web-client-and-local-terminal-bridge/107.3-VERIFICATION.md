## Lifecycle findings closed during verification

1. Explicitly closed ended sessions originally reappeared after reload. Close now removes the service record; interactive shells also remove themselves on natural exit, while configured commands retain ended output.
2. An in-flight workspace monitor rebuild could keep the CLI wrapper alive after HTTP shutdown. Disposal now detaches promptly, cannot publish after disposal, and has a blocked-rebuild regression test.
3. The web route initially formed a type-only circular dependency through `service/server.ts`. `SnapshotAdapter` now lives in a neutral service module and the dependency gate is clean.
4. A slow browser-transport shutdown could outlive the runtime's async SIGINT grace and leave a dead descriptor. Discovery is now withdrawn before socket/PTY teardown; isolated signal-stop and discovery lifecycle tests remove it automatically.
