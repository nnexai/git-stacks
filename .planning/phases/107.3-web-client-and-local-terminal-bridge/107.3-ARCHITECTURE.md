# Phase 107.3 Architecture

## Components

| Component | Responsibility |
|---|---|
| CLI `web` command | Start/discover the managed service, issue pairing, and open the loopback URL |
| Local service | Workspace authority, operations, signals, web security, and terminal processes |
| Web route adapter | Same-origin session admission and browser-safe DTO projection |
| Terminal manager | PTY lifecycle, replay, backpressure, visibility streaming, and process cleanup |
| Browser client | Workspace navigation, commands, signals, tabs, and xterm.js rendering |

## Control Plane

HTTP carries snapshots, creation catalogs, mutations, terminal metadata, priorities, and signal actions. SSE carries ordered workspace, operation, and signal changes with replay-gap recovery. Browser operations are filtered by principal ownership.

## Terminal Data Plane

Each terminal has one service-generated identity and one service-owned PTY. A WebSocket carries framed output plus input, resize, visibility, acknowledgements, title updates, and lifecycle controls. Hidden terminals disable bulk output transmission while their processes, signal filtering, and lifecycle controls remain active.

The manager retains bounded raw output with monotonically increasing cursors. Reattachment replays from the acknowledged cursor when available and sends an explicit reset/history-loss state when it is not.

## Security

- Loopback bind only.
- One-use pairing capability exchanged for a scoped HttpOnly cookie.
- Exact Host and Origin validation plus Fetch Metadata checks.
- No browser-visible bearer token, raw launch environment, or signal-ingress token.
- Stable-ID launch intent resolved inside the trusted service.
- Per-principal and global terminal quotas, bounded replay bytes, socket pressure limits, and retained-session expiry.

## Terminal Lifecycle

- Interactive shell exit removes the terminal tab.
- Configured command completion remains visible for result inspection.
- Explicit close terminates the process group, clears exact-surface activity, closes attachments, and deletes the retained session.
- Closing the browser detaches only the renderer; the terminal remains reconnectable while the service retains it.
- Managed-service shutdown terminates owned terminal processes.

## Signal Lifecycle

Authenticated OSC frames are recognized, stripped from terminal output, and published through the shared signal journal. Sidebar presence deduplicates providers; tabs preserve exact provider state. Activity is projected only for terminal surfaces still retained for that browser principal. Closing a surface publishes idle tombstones so future projections cannot resurrect stale work.
