## Findings and remediation

1. **Resolved — persistent signal dashboard:** The previous 300px right rail treated all signal history as equal persistent attention. Signals now open from compact header/workspace markers in a native-style popover with separate `Needs attention` and `Recent activity` groups.
2. **Resolved — incorrect signal semantics:** Working/completed activity no longer increments attention or exposes dismissal. Waiting, failed, and notification signals receive lifecycle-specific presentation; only notifications expose `Dismiss notification`.
3. **Resolved — shallow sidebar hierarchy:** The sidebar now uses Labels/Repositories organization, global Pinned/Active groups, exact workspace-repository rows, secondary branch state, semantic Git counters, provider badges, activity, unread markers, and hover/focus pin controls.
4. **Resolved — crowded contextual chrome:** Workspace open/close/copy actions moved behind a contextual action menu while `New shell` remains the primary visible action.
5. **Resolved — terminal bottom clipping:** Padding moved from the terminal host to the xterm element so FitAddon subtracts it when calculating rows and columns.

## Automated and live evidence

- Browser presentation tests cover attention grouping, active/background semantics, scope matching, provider/lifecycle labels, and relative time.
- Chromium at 1440×900 and 620×700 showed no document-level horizontal overflow.
- Live PTY resize changed `stty size` from `41 143` to `26 86`.
- At 620×700 the terminal host ended at y=700, xterm screen ended at y=684, and the configured 8px xterm inset remained fully visible; no terminal row crossed the available content edge.
- A real terminal OSC transition rendered `Codex needs your input`, exact workspace/repository/terminal context, `Needs input`, and `Focus terminal`; a subsequent completed transition removed it from attention.

## Remaining release-matrix checks

- Actual screen-reader passes, high-contrast mode, 200% browser text, RTL, and non-Chromium engines remain release-environment checks.
- The fixed narrow sidebar is intentionally still present at 620px; an overlay sidebar would be a separate adaptive-navigation enhancement rather than a blocker for native visual alignment.
