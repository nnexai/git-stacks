1. **Read all shard files.** List all `.json` files in `SHARDS_DIR` using `readdirSync`. For each file, `readFileSync` and `JSON.parse`. Wrap each parse in try/catch — skip malformed files with a warning log (defensive against partial writes from crashed subprocesses).

2. **Create and merge coverage map.** Create an empty map with `libCoverage.createCoverageMap({})`. For each parsed shard object, call `map.merge(shardData)`.

3. **Add zero-baseline entries for untouched files (D-07/D-08).** Load `join(COVERAGE_DIR, "blank-templates.json")` (written by the instrumentation step in Plan 02). Parse the JSON — it's a `Record<string, object>` keyed by original `src/...` paths. For each entry, check if the path already exists in `map.data`. If NOT, call `map.addFileCoverage(blankEntry)`. This ensures every `src/**/*.ts` file appears in the final report, even if no test touched it.

4. **Generate all five report formats in one pass (D-03/D-04).** Create a report context:
   ```typescript
   const context = libReport.createContext({
     dir: COVERAGE_DIR,
     coverageMap: map,
   ```

   - `reports.create("html").execute(context)` — writes `index.html` + per-file HTML pages
   - `reports.create("json").execute(context)` — writes `coverage-final.json`
   - `reports.create("json-summary").execute(context)` — writes `coverage-summary.json`
   - `reports.create("lcovonly").execute(context)` — writes `lcov.info`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.coverage/shards/*.json` → merge process | Per-subprocess JSON files are parsed and merged; malformed data could crash the merge |
| `blank-templates.json` → merge process | Instrumentation-generated blank templates are loaded and added to the coverage map |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-83-05 | T (Tampering) | `mergeAndReport()` shard parsing | mitigate | Wrap each shard `JSON.parse` in try/catch. Skip malformed shards with a warning log. Never crash the merge/report step due to one bad shard file. |
| T-83-06 | I (Information Disclosure) | `.coverage/` report artifacts | accept | Coverage reports reveal source structure and line-level hit counts. This is intentional for a local development tool. Reports are gitignored and never committed. Low risk — no PII or secrets in coverage data. |
</threat_model>
