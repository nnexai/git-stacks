**Step 2 — Create the minimal Istanbul smoke proof (only if Step 1 files are absent).** This is the Phase 82.1 Task 3 catch-up per D-09 prerequisite handling.

Create `tests/support/istanbul-smoke.ts` exporting an `instrumentAndRun` helper that:
1. Accepts a source file path and a subprocess command array.
2. Reads the source file, instruments it using `istanbul-lib-instrument`'s `createInstrumenter({ esModules: true, compact: false, parserPlugins: ["typescript"] })` and `instrumentSync(source, originalPath)`.
3. Writes the instrumented source to a caller-provided temp dir preserving relative structure.
4. Appends a `process.on("exit", ...)` shard writer to the instrumented entrypoint that serializes `globalThis.__coverage__` to `<shardDir>/<pid>-<timestamp>.json` using `writeFileSync` with `{ flush: true }`.
5. Returns the instrumented entrypoint path and shard directory path.

Create `tests/commands/istanbul-smoke.test.ts` that:
1. Uses `makeTmpDir` from `tests/helpers.ts` for a temp working directory.
2. Calls `instrumentAndRun` to instrument `src/commands/completion.ts` and `src/index.ts` (the minimal set needed for `completion bash`).
3. Runs `bun run <instrumented-index.ts> completion bash` as a subprocess using `Bun.spawnSync` with `GS_COVERAGE_SHARD_DIR` set to a temp shard dir.
4. Asserts the subprocess exits 0.
5. Reads the shard JSON from the temp shard dir.
6. Uses `istanbul-lib-coverage`'s `createCoverageMap` to load the shard and asserts at least one statement count is > 0 for a `src/` path.
7. Cleans up the temp dir in `afterEach`.
