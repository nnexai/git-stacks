<interfaces>
Existing mutation admission is `POST /v1/operations/{mutation}` followed by `OperationRegistry.submit({ clientId, endpoint, idempotencyKey, request, execution })`.

Existing discovery `ServiceLimitsSchema` contains transport limits only. Extend it with one native-model compatibility object matching the Zig storage contract: 16 workspaces, 16 labels per workspace, 8 repositories per workspace, 32 authoritative workspace/repository pairs, 32 distinct live pair identities, 32 reserved orphan tombstones, 16 surfaces per pair, 64 commands, and 64 attention items. The same object names UTF-8 byte ceilings for workspace name (96), workspace label (64), repository name (96), command ID (64), command name (96), surface command ID (64), surface title (64), surface cwd (128), attention ID (64), attention title (160), attention detail (500), and attention occurrence timestamp (40).

`createWorkspaceMutationAdapters()` already serializes open/close progress callbacks into operation reports. Extend this registry rather than creating a parallel operation store.
</interfaces>

| T-107-03-03 | Denial of Service | retries/concurrent creates | high | mitigate | Existing rate/deadline bounds, persistent idempotency, and atomic capacity reservation. |
