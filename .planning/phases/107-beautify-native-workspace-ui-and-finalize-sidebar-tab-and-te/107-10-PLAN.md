<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GTK form/events to worker | User input and repeated actions enqueue authenticated mutation/refresh work. |
| Worker threads to GTK main context | Owned decoded payloads cross thread/lifetime boundaries into widgets and model state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-107-10-01 | Repudiation | create double-submit/retry | high | mitigate | Controller freezes one submission/key, disables submit while active, and binds one operation ID. |
| T-107-10-02 | Denial of Service | synchronous GTK refresh | high | mitigate | Worker-owned transport/projection, coalesced queue, and callback-only main-thread projection. |
| T-107-10-03 | Tampering | stale/late worker callback | high | mitigate | Generation/cancellation checks plus owned dispatch memory and shutdown join/drain. |
| T-107-10-04 | Elevation of Privilege | native source selection | high | mitigate | Consume only authenticated catalog identities; never read YAML or accept caller paths. |
| T-107-10-05 | Denial of Service | terminal cleanup ordering | high | mitigate | Cancel/join workers before reverse-order UI/registry teardown and preserve registration-before-visible. |
</threat_model>
