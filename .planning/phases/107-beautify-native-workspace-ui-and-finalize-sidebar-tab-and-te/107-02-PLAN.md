<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller to creation domain | Untrusted name, branch, and source selectors enter engine-owned validation. |
| Creation plan to filesystem/Git | A validated plan creates directories, branches, worktrees, hooks, and YAML. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-107-02-01 | Tampering | name/branch/source validation | high | mitigate | Apply `NameSchema`, Git branch-format validation, registered-name lookup, and engine-derived paths before mutation. |
| T-107-02-02 | Elevation of Privilege | repository path selection | high | mitigate | Accept only registered repository identities and never caller-provided filesystem paths. |
| T-107-02-03 | Denial of Service | concurrent same-name create | medium | mitigate | Serialize by workspace name, recheck existence inside the lease, and release in `finally`. |
| T-107-02-04 | Repudiation | partial create rollback | medium | mitigate | Preserve progress plus rollback errors and commit YAML only after all tracked steps succeed. |
</threat_model>
