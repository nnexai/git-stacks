## Trust Boundaries

| Boundary | Description |
|----------|-------------|

| Hook subprocess to local service | Project hooks invoke an authenticated local endpoint using environment-derived routing identity. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-107-06-01 | Tampering | `.codex/hooks.json` merge/remove | high | mitigate | Validate before mutation, preserve unknown/unrelated content, match both publisher and source markers, and atomically replace. |
| T-107-06-02 | Elevation of Privilege | generated hook command | high | mitigate | Use validated workspace names, fixed executable/arguments, no raw filesystem paths, and an explicit user trust reminder. |
| T-107-06-03 | Information Disclosure | attention publication | medium | mitigate | Publish only provider/status/title/detail plus opaque workspace/repository/surface identities; reuse protected credential lookup. |
| T-107-06-04 | Denial of Service | absent or stalled local service | medium | mitigate | Best-effort mode propagates an AbortSignal with an injected bounded deadline, never auto-starts the service, and exits silently on expiry. |
