<interfaces>
`ProductionGraph` owns `model.State`, `service_client.Client`, and `tab_registry.Registry`. `tab_registry` is the exclusive terminal-host ownership authority; model navigation/refresh must not call teardown.

`reducer.reduce(state, .snapshot)` currently replaces the state and `app_graph.refreshSnapshot()` manually restores selected presentation. Replace the partial restore with one named reconciliation method.

Plan 107-07 reserves `State.orphan_tombstones[32]` separately from `State.pairs[32]` and names the 32-distinct-live-pair admission limit. Reconciliation owns tombstone population/release and must never append a tombstone to the authoritative pair array.

`service_sync.Coordinator` is UI-neutral: it returns intent/state values, while Plan 107-10 will own the actual worker thread and GTK main-context dispatch.
</interfaces>

| Boundary | Description |
|----------|-------------|
| Authoritative snapshot to native presentation | Engine-owned entities replace while process-owned hosts and user presentation must survive safely. |
| Worker completion/cursor to main model | Out-of-order asynchronous results can corrupt state or replay position. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-107-09-01 | Denial of Service | refresh trigger storms | high | mitigate | Single-flight generation, revision coalescing, and one bounded trailing refresh. |
| T-107-09-02 | Tampering | stale worker completion | high | mitigate | Generation IDs reject out-of-order completion and cursor adoption is success-atomic. |
| T-107-09-03 | Repudiation | vanished live terminal | high | mitigate | Separate labeled tombstone storage, bounded live-pair admission, registry ownership query, and one tested release path. |
| T-107-09-04 | Elevation of Privilege | asynchronous focus | high | mitigate | Reconciliation/refresh intents never emit platform focus; explicit user selection remains required. |
</threat_model>
