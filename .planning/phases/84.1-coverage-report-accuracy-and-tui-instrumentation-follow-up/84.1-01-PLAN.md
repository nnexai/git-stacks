<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test execution → coverage report | Passing tests are not sufficient; the report must show real source counters for known executed files. |
| preload instrumentation → normal test runner | Coverage instrumentation must stay opt-in and cannot slow or alter normal `bun run test*` behavior. |
| coverage artifacts → local gates | `.coverage/` files are untrusted until they are parsed and checked for sentinel hits. |
| CLI subprocesses → coverage shards | Child `bun run src/index.ts` executions must keep emitting mergeable Istanbul shards. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-84.1-01 | T | `scripts/coverage-preload.ts` | mitigate | Require a spike before final implementation and accept only a path that produces nonzero sentinel hits for regular TS and TUI TSX source. |
| T-84.1-02 | D | coverage runner scratch directories | mitigate | Run coverage commands serially because `.coverage/src-inst`, `.coverage/shards`, and `blank-templates.json` are shared mutable scratch paths. |
| T-84.1-03 | T | `.coverage/coverage-final.json` | mitigate | Add sentinel hit checks to `scripts/verify-gates.ts` so valid JSON with zero-hit known-executed files fails verification. |
| T-84.1-04 | I | public script surface | mitigate | Preserve existing coverage and test script names and artifact paths; do not introduce CI or threshold policy in this phase. |
</threat_model>
