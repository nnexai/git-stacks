# Phase 84.1 Plan 01: Coverage Instrumentation Repair Summary

Plan 01 is complete. The original two planned repair paths still failed, but internet research and local probes identified a third path that makes the report truthful: run coverage tests from an instrumented shadow root instead of trying to intercept original source imports at runtime.

## Research And Decision

- Bun docs confirm `--preload` loads setup before tests and runtime plugins can intercept module loading, but local probes showed the preload-only approaches still left both sentinels zero-hit.
- Bun docs also show built-in coverage can emit text/lcov, but this repo needs stable Istanbul JSON artifacts for `.coverage/coverage-final.json`, `.coverage/coverage-summary.json`, `.coverage/lcov.info`, and `.coverage/index.html`.
- Selected path: `instrumented runtime root`.

## Accomplishments

- Reproduced the defect: focused tests passed while `src/lib/messages.ts` and `src/tui/dashboard/ActionMenu.tsx` stayed zero-hit.
- Re-evaluated alternatives:
  - Redirect `onResolve`/`onLoad`: rejected because it still failed sentinel hits and missed relative-depth cases.
  - Load-time instrumentation in preload: rejected because shard collection still saw no counters.
  - Bun built-in coverage: rejected for this phase because it does not provide the required Istanbul JSON map contract.
  - Instrumented runtime root: accepted because both TS and TSX sentinels get nonzero statement/function hits.
- Updated `scripts/coverage-runner.ts` to create `.coverage/runtime-root`, copy tests/config, symlink shared runtime dependencies, and run `bun test` with `cwd: RUNTIME_ROOT`.
- Updated `scripts/coverage-preload.ts` to write test-process coverage shards with `afterAll(writeCoverageShard)`.
- Added verify-gates sentinel checks for `src/lib/messages.ts` and `src/tui/dashboard/ActionMenu.tsx`.
- Added regression tests for the runtime-root coverage mechanism and sentinel gate failures.
