**2. Create WizardView.tsx** — Reusable multi-step wizard component (per D-21, D-22, D-23).

Export a `WizardStep<T>` type:
```typescript
export type WizardStep<T> =
  | { kind: "text"; label: string; key: keyof T & string; prefill?: (data: Partial<T>) => string; validate?: (v: string) => string | undefined }
  | { kind: "confirm"; buildMessage: (data: Partial<T>) => string }
```

Export `WizardView` component with props:
```typescript
type Props<T extends Record<string, unknown>> = {
  steps: WizardStep<T>[]
  onComplete: (data: T) => void
  onCancel: () => void
}
```

Implementation details:
- Use `createSignal<number>(0)` for step index, `createSignal<Partial<T>>({})` for accumulated data — per D-22 (local signals).
- For `kind: "text"` steps: render `InlineInput` with `label={step.label}`, `prefill` computed from `step.prefill?.(data()) ?? ""`. On confirm, validate if step has `validate` — if validation returns an error string, show it as a `<text fg="red">` below the input and stay on the same step. On valid input, merge `{ [step.key]: value }` into data and advance step index. On cancel (escape from InlineInput), if step index > 0 decrement step index (per D-13), else call `props.onCancel()` (per D-14/D-15).
- For `kind: "confirm"` steps: render a `<box>` with `step.buildMessage(data())` as text (multiline, showing summary), plus `<text fg="gray">[y] Create  [Esc] Back</text>`. Use `useKeyboard` — on `y` call `props.onComplete(data() as T)`, on `escape` decrement step index.
- **Deferred focus pattern**: When step index changes (text step), mount `InlineInput` with a fresh `createSignal(false)` for focused, and `setTimeout(() => setFocused(true), 0)` — this prevents the keypress that triggered the step transition from leaking into the input (per Phase 11 pattern and CLAUDE.md rules).
- Wrap content in `<box flexDirection="column" paddingTop={1} paddingLeft={2}>`.
- Validation error display: below the InlineInput row, render `<Show when={validationError()}><text fg="red">  {validationError()}</text></Show>`.

**4. Fix useWorkspaces.ts reload()** — Change `reload` to return `Promise<void>`:
```typescript
function reload(): Promise<void> {
  const workspaces = listWorkspaces()
  const fresh: WorkspaceEntry[] = workspaces.map((ws) => ({
    workspace: ws,
    status: { state: "pending" as const },
  }))
  setEntries(fresh)
  setLoading(true)
  return fetchStatuses(workspaces, setEntries, () => cancelled).then(() => {
    if (!cancelled) setLoading(false)
  })
}
```
This is a one-line change: add `return` before the `fetchStatuses(...)` call and change the function signature. All existing callers that don't await continue to work (returning a promise they ignore is fine).
