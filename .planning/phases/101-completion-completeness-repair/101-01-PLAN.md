<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Commander metadata -> generated shell script | Command/option metadata is turned into shell code that users source into bash, zsh, and fish. |
| Config-root name lookups -> completion suggestions | Dynamic completion reads workspace/template/repo names from `~/.config/git-stacks` artifacts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-01 | Tampering | `src/lib/completion-generator.ts` | mitigate | Keep completion branches derived from the shared live Commander tree and existing fixed maps only; Task 2 repairs missing paths without introducing new unsanitized shell literals. |
| T-101-02 | Information Disclosure | dynamic lookup helpers | mitigate | Keep lookup scope limited to `workspaces/*.yml`, `templates/*.yml`, and `registry.yml`; Task 2 must not add broader filesystem or repo traversal during completion repair. |
| T-101-03 | Repudiation | live inventory source | mitigate | Task 1 centralizes command registration in `buildCliProgram()` so later tests and gates audit one canonical tree instead of a drifting duplicate. |
| T-101-SC | Tampering | Bun/Commander toolchain | mitigate | No new packages are introduced in this plan; verification uses existing `bun run test` and `bun run typecheck` only. |
</threat_model>
