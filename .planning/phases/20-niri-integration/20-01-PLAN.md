    **`open()` method — four sequential steps:**

    1. **NIRI_SOCKET gate (NIRI-08):** `if (!await isNiriRunning()) return null` — silent return, no spinner, no log.

    2. **Create/reuse named workspace (NIRI-01, NIRI-04):**
       - Start spinner: `"Setting up niri workspace"`
       - `const workspaceName = ctx.workspace.name`
       - `const workspaces = await listNiriWorkspaces()`
       - `const alreadyNamed = workspaces.some(ws => ws.name === workspaceName)`
       - If NOT alreadyNamed: `await setNiriWorkspaceName(workspaceName)` (names the currently active workspace)
       - `await focusNiriWorkspace(workspaceName)` — always called (both first-open and re-open)

    3. **Move prior integration windows (NIRI-02):**
       - `const niriWindows = await listNiriWindows()`
       - Iterate `Object.values(bag)`, filter for `artifact?.kind === "window"`
       - For each WindowArtifact, find matching niri window: `niriWindows.find(w => w.pid != null && w.pid === artifact.pid)`
       - CRITICAL: Guard `w.pid != null` because NiriWindow.pid is `number | null | undefined`
       - If match found: `await moveWindowToWorkspace(match.id, workspaceName)` wrapped in try/catch
       - On move failure: `p.log.warn(...)` and continue (do NOT throw — partial failure is acceptable)

    4. **Run user-configured commands (NIRI-03/NIRI-09 replacement):**
       - Parse config: `const config = niriConfigSchema.parse(ctx.config.integrations["niri"] ?? {})`
       - If `config.commands?.length`: construct hookEnv with `WS_WORKSPACE`, `WS_BRANCH`, `WS_TASKS_DIR`
       - Call `await runHooks(config.commands, ctx.tasksDir, hookEnv, false)` — abortOnFailure=false
       - Stop spinner: `"niri workspace ready"`

    **Error handling:** Wrap the entire body (after NIRI_SOCKET gate) in try/catch. On error: `spinner.stop("niri unavailable -- skipped")`, `p.log.warn(...)`, return null.

    **Return value:** Always `null` — tier-3 integrations are consumers, not producers. No NiriArtifact type needed.

    **`configurePrompt()`:** Return `{ enabled: true }` — no interactive prompts for commands array in v0.6.0.

    **NIRI-05 note:** No cleanup on remove — this is intentionally not implemented per user decision. Do NOT add any remove/cleanup method.
