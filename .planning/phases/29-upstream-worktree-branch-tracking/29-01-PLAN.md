<feature>
  <name>Upstream Branch Tracking Detection and Setup</name>
  <files>src/lib/git.ts, tests/lib/git.test.ts</files>
  <behavior>
    ## checkRemoteTrackingRef(repoPath, branch) -> Promise&lt;boolean&gt;
    - Returns true when `origin/<branch>` ref exists in local remote-tracking refs (e.g., after a fetch)
    - Returns false when `origin/<branch>` does not exist locally
    - Uses `git -C ${repoPath} rev-parse --verify origin/${branch}` with `.quiet().nothrow()`

    ## checkBranchExistsOnRemote(repoPath, branch) -> Promise&lt;boolean&gt;
    - Returns true when branch exists on the remote origin (network call)
    - Returns false when branch does not exist on remote
    - Uses `git -C ${repoPath} -c fetch.timeout=10 ls-remote --exit-code --heads origin ${branch}` with `.quiet().nothrow()`
    - The `-c fetch.timeout=10` prevents hanging when offline (per research open question)

    ## hasUpstreamTracking(repoPath, branch) -> Promise&lt;boolean&gt;
    - Returns true when `branch.<name>.remote` git config is set (branch has upstream configured)
    - Returns false when no upstream tracking is configured
    - Uses `git -C ${repoPath} config branch.${branch}.remote` with `.quiet().nothrow()`

    ## ensureUpstreamTracking(repoPath, branch) -> Promise&lt;{ tracked: boolean; source?: "local" | "remote" }&gt;
    - If already tracked (hasUpstreamTracking returns true): returns `{ tracked: false }` immediately
    - If local remote-tracking ref exists (checkRemoteTrackingRef): sets tracking and returns `{ tracked: true, source: "local" }`
    - If local ref missing but branch exists on remote (checkBranchExistsOnRemote): sets tracking and returns `{ tracked: true, source: "remote" }`
    - If branch does not exist on remote: returns `{ tracked: false }` (brand-new branch, no upstream)
    - If ls-remote fails (network error): returns `{ tracked: false }` (non-fatal, skip tracking)
    - Tracking is set via `git -C ${repoPath} branch --set-upstream-to=origin/${branch} ${branch}` with `.quiet().nothrow()`
  </behavior>
  <implementation>
