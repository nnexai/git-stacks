# ADR 108: Package Ownership and Transitional Entrypoints

## Decision

The repository is an npm workspace for publishable protocol, client, core, CLI, service, web, and TUI packages. The public `git-stacks` facade depends on the CLI package and provides the default executable. Node 24 is the minimum machine-side runtime. Bun is isolated to the optional TUI build and renderer tests; no Bun production CLI, service shim, general test runner, or coverage runner remains.

The TUI participates in workspace development but is absent from the default consumer dependency graph. It declares Bun/OpenTUI explicitly and is installed only when a user chooses the optional `@git-stacks/tui` package.

Project source is bundled per package by esbuild while declared dependencies remain external. This avoids Node ESM extension ambiguity during migration without embedding third-party dependency trees.

## Ownership

- `protocol`: browser-safe schemas, IDs, DTOs, and version negotiation.
- `client`: carrier interfaces and shared event/presentation reducers.
- `core`: domain behavior, persistence, and injected machine capabilities.
- `cli`: local Commander/prompt presentation and the public daemonless command.
- `service`: Node authority, operations, projections, transport, signals, and PTYs.
- `web`: browser-only rendering and same-origin carriers.
- `tui`: optional Bun/OpenTUI rendering, trusted service client, shared pure core projections/types, and explicit foreground handoffs. It does not perform authoritative reads or writes directly.

## Cutover result

The transitional shim inventory is empty. Canonical production source lives under `packages/*/src`; the former root `src/` tree and Bun CLI/service entrypoints were removed in the Phase 115 cutover.
