**describe("hook env injection")** — Uses probe hooks that write env vars to an artifact file, then asserts on the file content. Per D-08, prefer artifact-based proof over stdout-only.

1. **"post_open hook receives GS_WORKSPACE_NAME and GS_WORKSPACE_BRANCH"** — Create workspace fixture with a repo. Use `writeProbeScript(tmpDir, "probe-open.sh", probeFile, ["GS_WORKSPACE_NAME", "GS_WORKSPACE_BRANCH", "GS_TRIGGERED_BY"])` and set workspace `hooks.post_open: [scriptPath]`. Run `runCli(cfgDir, ["open", wsName, "--no-ide"])`. Read `probeFile` and assert it contains `GS_WORKSPACE_NAME=<wsName>` and `GS_WORKSPACE_BRANCH=feat/<wsName>` and `GS_TRIGGERED_BY=open`.

2. **"post_open hook receives GS_REPO_NAME and GS_REPO_PATH per repo"** — Similar setup with a probe that captures `GS_REPO_NAME` and `GS_REPO_PATH`. Assert the probe file contains correct repo name and the absolute task_path.

3. **"pre_close hook fires during close command"** — Set `hooks.pre_close: [scriptPath]` with a probe script. Run `runCli(cfgDir, ["close", wsName])`. Assert probe file was created and contains workspace name.

**describe("hook cwd correctness")** — Proves hooks run in the correct directory.

4. **"hook runs with cwd set to repo task_path"** — Probe script writes `PROBE_PWD=$(pwd)` to artifact. After running `open --no-ide`, read artifact and verify `PROBE_PWD` matches the repo's absolute task_path.

5. **"hook cwd is absolute, not relative"** — Same as above but additionally verify the `PROBE_PWD` value starts with `/`.

**describe("explicit cwd/path handling")** — Proves commands work correctly regardless of the shell cwd where the CLI is invoked. This is the SC 6 proof.

6. **"status command produces correct results when invoked from /tmp"** — Run `runCli(cfgDir, ["status", wsName, "--json"], { cwd: "/tmp" })`. Verify JSON output is correct (same as when run from project root). This proves the command uses explicit paths from config, not shell cwd.

7. **"env command produces correct results when invoked from a different directory"** — Run `runCli(cfgDir, ["env", wsName, "--format", "json"], { cwd: "/tmp" })`. Verify env JSON contains correct `GS_WORKSPACE_NAME`, `GS_WORKSPACE_BRANCH`, etc.

8. **"run command executes in repo cwd regardless of CLI invocation directory"** — Create workspace fixture, then run `runCli(cfgDir, ["run", wsName, "--", "pwd"], { cwd: "/tmp" })`. The stdout should contain the repo's task_path (proving `run` uses repo path, not shell cwd).

9. **"paths command returns absolute paths regardless of CLI cwd"** — Run `runCli(cfgDir, ["paths", wsName], { cwd: "/tmp" })`. Verify stdout paths are absolute and match the fixture's task_path.

**describe("env file generation")** — Proves env files are written to repos.

10. **"open writes env file to repo when env_file configured"** — Create workspace fixture with `envFile: ".env"` and `env: { MY_VAR: "hello" }`. Run `open --no-ide`. Verify `.env` file exists at repo task_path and contains `MY_VAR=hello`.

Every assertion failure should provide enough context per D-09. Assert `exitCode === 0` and `stderr === ""` (or check stderr for expected content) before asserting on stdout/artifacts, so failures show the CLI error output.
