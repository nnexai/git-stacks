<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| live CLI registrations → canonical inventory | The shipped command tree must be compared against the single approved inventory source without stale manual duplication. |
| canonical inventory → mapped test files | Inventory metadata references filesystem paths that may drift or disappear. |
| `.coverage/` artifacts → local gate decision | Machine-readable coverage files are untrusted inputs until parsed and validated. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-84-03 | T | `scripts/verify-gates.ts` inventory comparison | mitigate | Derive the live command surface from current Commander registrations and compare it directly to the canonical Phase 80 inventory instead of a second hand-maintained list. |
| T-84-04 | D | aggregated gate execution | mitigate | Collect all inventory, mapping, and coverage findings before exiting so maintainers fix drift in one pass rather than repeated fail-fast reruns. |
| T-84-05 | T/I | `.coverage/` artifact parsing | mitigate | Parse only the stable Phase 83 `.coverage/` outputs, reject missing/invalid files explicitly, and avoid threshold or CI policy expansion beyond the locked local-only contract. |
</threat_model>
