<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Template YAML -> composed template object | Stored template label data from multiple files is merged into one in-memory template before workspace creation. |
| Source workspace YAML -> cloned workspace YAML | Existing persisted labels are copied into a new persisted workspace document. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-04 | T | `src/lib/composition.ts` label merge path | mitigate | Merge only validated `Template.labels` values from `readTemplate()` outputs, preserve deterministic order, and deduplicate so composed templates cannot multiply or synthesize labels outside persisted input. |
| T-74-05 | I | `src/tui/workspace-clone.ts` clone snapshot | mitigate | Copy `source.labels` directly into the new workspace object and avoid rereading template state, which preserves the snapshot semantics required by ASVS-style data integrity expectations for local config writes. |
| T-74-06 | D | `tests/lib/composition.test.ts` and `tests/tui/workspace-clone.test.ts` | mitigate | Add regression tests for includes, multi-template composition, and clone flows so missing-label regressions block the change before release. |
</threat_model>
