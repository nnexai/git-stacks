## Shared Patterns

### Pure projection, thin GTK

Apply to `workspace_view.zig`, `attention_view.zig`, and `app.zig`: all classification, priority, grouping, compression, copy, and accessibility-description generation is deterministic Zig without GTK. GTK binds semantic outputs and platform settings. A planner task that adds domain inference inside `appendPairButton()` or `refreshAttentionRows()` violates the architecture.

### Stable identity and bounded state

Apply end-to-end: service status maps to repository IDs; native projection keys are `PairKey`; signal actions carry final service signal identity and exact workspace/repository/surface/session identity. Validate capacities before copying and add compile-time/model-contract drift coverage.

### Optional capability semantics

PR/check absence means omit the badge. Missing/degraded repository state is explicit and text-redundant. No layer below GTK should manufacture unavailable state, and GTK must not probe providers or Git.

### Accessibility and selection dominance

Use GTK accessible properties/states beside visible symbolic icons and text. Selection remains a separate semantic state and neutralizes decorative accents. Awaiting, activity, and unread remain distinct in visible and accessible output; no meaning relies on color or motion alone.

### Error and action handling

Reducers validate stale identities and return explicit effects/fallback reasons. GTK keeps failed dismiss rows visible and exposes retry; focus failure falls back to workspace/repository when valid. Rendering never marks read or dismissed merely because a row was displayed.
