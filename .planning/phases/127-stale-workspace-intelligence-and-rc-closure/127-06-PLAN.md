<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Protocol response → shared presentation | Finite authority becomes user-facing copy and ordering. |
| Concurrent RPC/reload → client state | Late or conflicted responses could replace newer authoritative state. |
| Keyboard events → action IDs | Input composition/focus/repeats could trigger unintended commands. |
| TUI package → trusted service client | Service access must remain isolated to the approved adapter seam. |

| T-127-06-01 | Spoofing | Shared labels/presentation | medium | mitigate | Exhaustive fixed maps, exact UTC, neutral wording, and no local classification/score. |
| T-127-06-02 | Tampering | Generation/conflict coordinator | high | mitigate | Monotonic tokens, expected revision equality, one reload/retry, and late-response tests. |
| T-127-06-03 | Elevation of Privilege | Shared action coordination | high | mitigate | Delegate only to existing canonical one-shot descriptors; no local inventory construction. |
| T-127-06-04 | Spoofing / Repudiation | Shortcut matching | medium | mitigate | Physical codes, exact modifiers, composition/AltGraph guards, scope metadata, and repeat suppression. |
| T-127-06-05 | Elevation of Privilege | TUI official-service boundary | high | mitigate | Static package checks and one isolated `@git-stacks/service/client` import seam. |
