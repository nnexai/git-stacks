<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Secure request → authoritative core state | Untrusted expected revision/force input requests potentially expensive evidence. |
| Evaluator internal result → browser/trusted clients | Internal fields and provider-adjacent data must be allowlisted and bounded. |
| Managed service lifecycle → volatile cache | Cache scope must not outlive or leak between service instances. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-127-05-01 | Tampering | Secure route revision | high | mitigate | Strict request parse and core revision equality occur before evaluator/cache/probes; tests assert zero calls on conflict. |
| T-127-05-02 | Information Disclosure | Web projection | high | mitigate | Field-by-field allowlist plus final strict schema parse and disclosure canaries. |
| T-127-05-03 | Denial of Service | Force-refresh route | medium | mitigate | Existing authenticated scope, abort propagation, evaluator bounded fan-out, and no background polling. |
| T-127-05-04 | Elevation of Privilege | Router evaluator interface | high | mitigate | Narrow read-only interface, `snapshot.read` method, no lifecycle dependencies or durable operation path. |
| T-127-05-05 | Information Disclosure | Service cache lifetime | medium | mitigate | Per-managed-service construction, memory-only entries, and cold-cache restart tests. |

**ASVS L1 gate:** revision, projection, and confused-deputy threats are high severity and must be green before adapters or UI consume the route.
</threat_model>
