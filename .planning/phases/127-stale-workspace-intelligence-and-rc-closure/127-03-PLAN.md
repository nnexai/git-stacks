<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Persisted source metadata → command argv | Workspace-controlled strings influence an external provider CLI invocation. |
| Provider process output → core result | Untrusted, credential-adjacent JSON and errors are parsed into application evidence. |
| Core result → browser protocol | Only bounded, sanitized evidence may cross to interactive clients. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-127-03-01 | Tampering / Elevation | GitHub/GitLab argv builders | high | mitigate | Complete provenance validation, fixed executable/flags, separate variables, repository encoding, and no shell/search/mutation. |
| T-127-03-02 | Information Disclosure | Provider parser and DTO schemas | high | mitigate | Parse allowlisted fields only; fixed unknown enums; strict path/raw-output/credential-free protocol schemas. |
| T-127-03-03 | Denial of Service | Provider subprocess | medium | mitigate | Reused timeout, abort, process cleanup, maximum output bytes, and strict JSON bounds. |
| T-127-03-04 | Spoofing | Provider/change identity | high | mitigate | Provider/type/host/repository/number must match validated persisted provenance; no branch/remote inference. |
| T-127-03-05 | Tampering | Provider timestamps | medium | mitigate | Validate ISO timestamps; malformed provider data becomes unknown rather than confirmed evidence. |

**ASVS L1 gate:** the high-severity injection, disclosure, and identity threats must be green in `forge-change-status.test.ts`; unresolved high severity blocks Plan 127-04.
</threat_model>
