<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Official service response → TUI UIView | Revisioned evidence enters a trusted interactive renderer. |
| Terminal keyboard → stale/global handlers | Key ownership and small-screen fallbacks must prevent command leakage. |
| Selected stale row → lifecycle inventory | Advisory rows request but cannot create mutation authority. |
| Long provider/workspace text → OpenTUI renderer | Untrusted text must remain bounded, non-nested, and disclosure-safe. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-127-08-01 | Tampering | TUI response generations | high | mitigate | Shared gate, revision equality, one retry, and late-response isolated Bun tests. |
| T-127-08-02 | Elevation of Privilege | `a` lifecycle bridge | high | mitigate | Candidate-only access, fresh canonical inventory, typed Force sequence, and no local descriptors. |
| T-127-08-03 | Spoofing | Key routing | medium | mitigate | Active-view-first handling, exact key table, pending suppression, and Escape-only too-small branch. |
| T-127-08-04 | Denial of Service | Narrow/short/long rendering | medium | mitigate | Exact tiers, bounded protocol arrays, clamped selection/scroll, safe fallback, and no nested text. |
| T-127-08-05 | Information Disclosure | TUI text/error output | high | mitigate | Shared fixed copy, strict response, disclosure canaries, and no service/core/provider raw imports. |
| T-127-08-06 | Repudiation | Operation reconnect/result | medium | mitigate | Existing durable operation IDs and typed terminal results; reconciliation never resubmits. |

**ASVS L1 gate:** generation, action authority, and disclosure threats are high severity and must pass the isolated TUI suite before cross-client closure.
</threat_model>
