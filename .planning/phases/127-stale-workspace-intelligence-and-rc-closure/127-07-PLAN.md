<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Secure browser RPC → overlay state | Revisioned sanitized evidence enters long-lived retained UI state. |
| Keyboard/pointer → overlay/global navigation | Context leakage or repeats could trigger unintended refresh/Open/lifecycle actions. |
| Overlay row → canonical lifecycle service | Advisory evidence must not become local authorization. |
| Service text → DOM/accessibility attributes | Long/untrusted text must remain bounded and disclosure-safe. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-127-07-01 | Tampering | Retained response generations | high | mitigate | Shared monotonic gate, revision equality, one conflict retry, and retained-data race tests. |
| T-127-07-02 | Elevation of Privilege | Candidate action menu | high | mitigate | Fresh service descriptors, incomplete denial, typed dirty sequence, exact-name confirmation, and no local descriptor creation. |
| T-127-07-03 | Spoofing | Stale presentation | medium | mitigate | Shared fixed labels/service order and explicit candidate/incomplete/unknown/caution sections. |
| T-127-07-04 | Information Disclosure | DOM, title, error, long text | high | mitigate | Strict projection, shared fixed errors, canary tests, and no hidden sensitive attributes. |
| T-127-07-05 | Denial of Service | Repeats, overflow, long collections | medium | mitigate | Pending suppression, bounded protocol arrays, vertical scroll, responsive wrapping, and no horizontal overflow. |
| T-127-07-06 | Repudiation | Focus and action completion | low | accept | Existing overlay/action result semantics plus exact focus restoration provide sufficient L1 traceability; no durable audit is introduced. |
