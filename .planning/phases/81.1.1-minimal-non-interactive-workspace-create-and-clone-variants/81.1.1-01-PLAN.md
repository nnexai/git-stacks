## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → TUI function | Untrusted user input (name, branch, template identifiers) crosses into workspace creation/clone logic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81.1.1-01 | Tampering (V5) | `--name <new-name>` on clone | mitigate | `validateName(opts.name)` in workspace.ts clone action handler applies NameSchema (Zod) — same validation as `new [name]` positional arg; rejects names with path traversal, special chars, or invalid length |
| T-81.1.1-02 | Tampering (V5) | `--from <value>` in non-interactive new | mitigate | Non-interactive guard resolves `--from` ONLY via `templateExists()` (config lookup), never via `existsSync()` + filesystem operations; local-path `--from` is inert in non-interactive mode per D-10 scope boundary |
| T-81.1.1-03 | Spoofing | `--non-interactive` flag bypass | accept | Flag is a boolean opt-in; no authentication or authorization involved; same commands available interactively |
| T-81.1.1-04 | DoS | Rapid automated invocation | accept | Same risk profile as interactive mode; no amplification vector; workspace creation is idempotent-safe (fails on duplicate name) |
</threat_model>
