## Decisions Made

- Unit tests run in single shared `bun test` process — sufficient because they use mock.module, not real I/O
- Integration tests run per-file in isolated processes — prevents mock.module cache contamination between test files
- Mock factories use `Record<string, unknown>` return type with `overrides` spread at end — enables targeted override without specifying all props
- Schema stubs use structural `{parse, safeParse}` mocks rather than actual Zod schemas — avoids importing real Zod in test setup
