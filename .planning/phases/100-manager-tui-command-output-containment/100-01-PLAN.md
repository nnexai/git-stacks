<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Non-editor commands inherit stdout/stderr and corrupt the alternate-screen UI | high | Add captured shell execution that pipes both streams and expose it through manual command callback wiring. |
| CLI behavior regresses because manual commands always switch to captured mode | medium | Gate captured execution on `opts.onOutput`; keep existing `runShellSequence()` path when absent. |
| Captured output grows without bound during noisy commands | medium | Add bounded output helper with a fixed limit and omitted-count tracking. |
</threat_model>
