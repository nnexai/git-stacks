<objective>
Switch workspace and template shell completion helpers from filename-based listing to YAML `name:` field extraction.

Purpose: Phase 33 established that workspaces and templates are identified by their YAML `name` field, not by filename. The shell completion helpers still use `ls *.yml | sed 's/.yml$//'` (bash/fish) or glob `*.yml(N:t:r)` (zsh) to list candidate names. This means completions show filenames — which may differ from the actual `name` field stored in YAML. This plan fixes all three shells to extract `name:` from YAML files using grep/sed.

Output: Updated `completion-generator.ts` with YAML-based name extraction for workspace and template helpers in all three shells; tests proving the new extraction method is used.
</objective>

**Target pattern for workspace/template extraction (all shells):**
```bash
# grep the YAML name: field from all .yml files in the directory
grep -h '^name:' "$DIR"/*.yml 2>/dev/null | sed 's/^name:[[:space:]]*//'
```

The `-h` flag suppresses filename prefixes from grep output. The `[[:space:]]*` handles optional whitespace after the colon. The `2>/dev/null` suppresses errors when the directory is empty or missing.
