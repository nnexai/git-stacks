<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service projection → browser memory | Trusted host state is reduced before crossing into untrusted browser presentation. |
| Browser input → operation.submit | Pointer/keyboard/typed confirmation becomes a revision-bound destructive intent. |
| Authoritative reload → local views | Replacement sets invalidate stale selected IDs, tabs, signals, and counts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-123-05 | Information Disclosure | `projectWebSnapshot` / lifecycle projection | high | mitigate | Exact allowlist, minimal archived schema, active-ID signal filtering, and encoded negative tests for paths/secrets/terminal metadata. |
| T-123-06 | Tampering | stale/reconnected browser lifecycle intent | high | mitigate | Expected revision, fresh idempotency key per confirmed intent, clear-on-conflict, refresh, and no automatic destructive resubmit. |
| T-123-09 | Elevation of Privilege | force confirmation UI | high | mitigate | Dirty-only typed eligibility, exact-name disabled state, fresh authoritative refresh, and independent server validation from Plan 04. |
| T-123-10 | Tampering | successor ordering | medium | mitigate | Consume the shared pin/priority/activity/name/ID comparator and prove every tier independently after authoritative refresh. |
| T-123-13 | Repudiation | removal progress/failure presentation | medium | mitigate | Render service-owned named phases and typed terminal/blocker status; do not infer completion from optimistic UI. |
</threat_model>
