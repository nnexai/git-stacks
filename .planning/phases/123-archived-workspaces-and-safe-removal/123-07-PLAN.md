|----------|-------------|
| Keyboard/input → TUI state machine | User input chooses reversible or destructive lifecycle intent. |
| TUI → trusted service client | Stable ID, revision, and exact-name confirmation cross into service authority. |
| Service events/core reload → rendered state | Typed progress/failures and authoritative replacement state drive selection and views. |

| T-123-09 | Elevation of Privilege | TUI force-name flow | high | mitigate | Dirty-only discriminated view, exact-name input, new intent, and server validation from Plan 04. |
