|----------|-------------|
| Workspace YAML → active/archive catalog | Persisted activity and archive state become authoritative projections. |

| Client lifecycle input → strict mutation schema | Revision-bound intent and force-only confirmation enter service authority. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |

| T-123-06 | Tampering | lifecycle mutation schema | high | mitigate | Stable ID/revision strict union and force-only confirmation field with extra-field rejection. |
| T-123-10 | Tampering | successor ordering | medium | mitigate | Service-carried persisted activity and one comparator with independent tie-tier tests in web and TUI plans. |

</threat_model>
