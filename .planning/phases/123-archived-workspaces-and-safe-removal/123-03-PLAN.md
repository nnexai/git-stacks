<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lifecycle admission → PTY allocation | A target mutation must prevent a new terminal from entering after the close barrier begins. |
| Terminal manager → operating-system process | Signal delivery is only an attempt; exit settlement is the completion proof. |
| Internal workspace close → public terminal API | Cross-principal cleanup authority must not become client-visible enumeration authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-123-02 | Tampering / Denial of Service | target admission | high | mitigate | Stable-ID target gate checked before PTY allocation with unrelated-target concurrency tests. |
| T-123-07 | Tampering / Data Loss | PTY close escalation | high | mitigate | Shared close promise plus observed TERM/KILL exit; second timeout is cleanup_failed. |
| T-123-12 | Information Disclosure | internal workspace close | medium | mitigate | Keep cross-principal enumeration internal and return bounded counts/status only. |
</threat_model>
