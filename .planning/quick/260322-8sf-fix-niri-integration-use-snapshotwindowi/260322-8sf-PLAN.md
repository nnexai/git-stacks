```typescript
export async function runIntegrationCleanup(ctx: IntegrationContext): Promise<void> {
  const sorted = [...integrations].sort((a, b) => a.order - b.order)
  for (const integration of sorted) {
    if (!integration.isEnabled(ctx)) continue
    if (integration.applies && !integration.applies(ctx.workspace)) continue
    if (!integration.cleanup) continue
    try {
      await integration.cleanup(ctx)
    } catch (err) {
      // Cleanup failures are non-fatal — log and continue
      console.warn(`${integration.id} cleanup: ${String(err)}`)
    }
  }
}
```

**src/lib/integrations/vscode.ts — use snapshotWindowIds for niri window capture:**

Add import at top:
```typescript
import { isNiriRunning, snapshotWindowIds } from "../niri"
```

In the `open()` method, replace the current Bun.spawn block (the try block starting around line 37) with:
```typescript
try {
  const niriActive = await isNiriRunning()
  let niriWindowIds: number[] | undefined

  if (niriActive) {
    // Use snapshot-diff to capture niri window IDs — PID matching fails for Electron forks
    niriWindowIds = await snapshotWindowIds(async () => {
      Bun.spawn([cmd, artifactPath], {
        stdout: "ignore",
        stderr: "ignore",
        stdin: "ignore",
      })
    })
    if (niriWindowIds.length === 0) niriWindowIds = undefined
  } else {
    Bun.spawn([cmd, artifactPath], {
      stdout: "ignore",
      stderr: "ignore",
      stdin: "ignore",
    })
  }

  const app_id = cmd.split("/").at(-1) ?? cmd
  // pid is 0 when captured via snapshot (PID unreliable for Electron), kept for non-niri consumers
  return { kind: "window", pid: 0, app_id, title: "", niriWindowIds }
} catch {
  return null
}
```

Wait — we still want pid for potential non-niri consumers. But when using snapshotWindowIds, the spawn happens inside the callback and we don't have access to proc.pid. Two options: (a) always use snapshotWindowIds and accept pid=0 when niri is active, or (b) spawn normally, capture pid, then also do snapshotWindowIds.

Option (b) is wrong — snapshotWindowIds needs to wrap the spawn to diff before/after.

Option (a) is correct. When niri is active, pid is unreliable anyway (Electron forks). Set pid to 0 and rely on niriWindowIds. When niri is NOT active, spawn normally and return pid as before.

Revised approach for the niri-active branch:
```typescript
if (niriActive) {
  let spawnedPid = 0
  niriWindowIds = await snapshotWindowIds(async () => {
    const proc = Bun.spawn([cmd, artifactPath], {
      stdout: "ignore", stderr: "ignore", stdin: "ignore",
    })
    spawnedPid = proc.pid
  })
  if (niriWindowIds.length === 0) niriWindowIds = undefined
  const app_id = cmd.split("/").at(-1) ?? cmd
  return { kind: "window", pid: spawnedPid, app_id, title: "", niriWindowIds }
} else {
  const proc = Bun.spawn([cmd, artifactPath], {
    stdout: "ignore", stderr: "ignore", stdin: "ignore",
  })
  const app_id = cmd.split("/").at(-1) ?? cmd
  return { kind: "window", pid: proc.pid, app_id, title: "" }
}
```

This keeps pid populated (best-effort) AND adds niriWindowIds when niri is active.
