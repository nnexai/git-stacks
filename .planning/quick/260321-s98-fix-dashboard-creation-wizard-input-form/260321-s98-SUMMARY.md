# Quick Task 260321-s98: Fix Dashboard Creation Wizard Input Form Summary

**One-liner:** Removed double-nested CenteredDialog in WizardView text steps so wizard input spans the full medium dialog width instead of a clipped ~35%.

## What Was Done

WizardView wrapped its content in `CenteredDialog(size="medium")` and then rendered `InlineInput` which internally wraps in another `CenteredDialog(size="small")`. This double-nesting caused:

- Input width = 50% of 70% = ~35% of terminal width
- Inner dialog border visually clipped the label and input
- All three wizard flows (create workspace from template, create workspace from repos, create template) were affected since they all use WizardView

The fix renders the label `<text>` and `<input>` directly inside WizardView's own CenteredDialog, eliminating the inner dialog entirely. Escape key handling for text steps was moved into WizardView's `useKeyboard` callback (previously delegated to InlineInput's internal handler).

InlineInput.tsx was not modified — it is still used standalone in App.tsx where its own dialog wrapping is correct behavior.
