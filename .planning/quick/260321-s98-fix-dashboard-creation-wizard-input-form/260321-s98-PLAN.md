<objective>
Fix the dashboard creation wizard's text input forms clipping outside the dialog and showing a short visible input width.

Purpose: The WizardView renders InlineInput for text steps, but InlineInput wraps itself in its own CenteredDialog (size="small"). Since WizardView already wraps content in a CenteredDialog (size="medium"), this creates a double-nested dialog: a small dialog inside a medium dialog. The inner dialog's 50% width of the outer dialog's 70% means the input gets only ~35% of terminal width, and the inner border clips content.

Output: WizardView renders the input element and label directly instead of delegating to InlineInput, eliminating the double-nesting.
</objective>

      }
