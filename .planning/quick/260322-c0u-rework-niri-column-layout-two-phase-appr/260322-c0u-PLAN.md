<objective>
Rework niri integration from inline spawn-and-position to a two-phase approach: (1) create all windows and collect a windowId map, (2) reorder, stack, apply widths, and focus using the map. This eliminates reliance on window focus for ordering, which breaks under focus-follows-mouse.

Purpose: The current approach interleaves spawning with positioning using `focusNiriWindow` + `setNiriColumnWidth`, which races with focus-follows-mouse. The two-phase approach spawns everything first, then uses `move-column-to-index` (1 IPC call per column) and `set-window-width --id` (focus-independent) for reliable layout.

Output: Updated `src/lib/niri.ts` with new wrappers and schema, reworked `src/lib/integrations/niri.ts` with two-phase open(), all tests passing.
</objective>

  <action>
    Restructure the open() method in src/lib/integrations/niri.ts into two distinct phases:

    **Phase 1 — Creation (collect windowId map):**

    After Step 2 (bag window moves, unchanged), process all columns but ONLY spawn/resolve windows. Do NOT stack, apply widths, or reorder during this phase.

    Build a `columnMap: Map<number, number[]>` where key = 0-based column config index, value = window IDs in that column (in config order).

    For each column (index `ci`), for each window config:
    - `source:` — resolve windowId from bag (same as current). Push to `columnMap.get(ci)`.
    - `app:` — snapshotWindowIds + niriSpawn (same as current). Push returned IDs to `columnMap.get(ci)`.
    - `command:` — snapshotWindowIds + niriSpawnSh (same as current). Push returned IDs to `columnMap.get(ci)`.
    - Track `focusWindowId` if `window.focus === true` (same as current).

    Do NOT call consumeOrExpelWindowLeft, focusNiriWindow, setNiriColumnWidth, or setWindowWidth during Phase 1.

    **Phase 2 — Layout (reorder, stack, width, focus):**

    Only run Phase 2 if `columnMap.size > 0`.

    **Step 2a — Reorder columns left-to-right:**
    Process target columns in order (index 0, 1, 2...). For each target column index `ci`:
    - Get the first windowId from `columnMap.get(ci)`. Skip if empty.
    - Call `focusNiriWindow(windowId)` then `moveColumnToIndex(ci + 1)` (1-based index).
    - This places the column at the correct position. Already-placed columns (to the left) are not displaced because we process left-to-right.

    **Step 2b — Stack windows within columns:**
    For each column in columnMap, if it has > 1 window, call `consumeOrExpelWindowLeft(windowId)` for windows at index 1+ (same as current stacking logic, just deferred).

    **Step 2c — Apply widths:**
    For each column config with a `width` value and at least one window in columnMap:
    - Call `setWindowWidth(firstWindowId, column.width)` — this uses the new --id variant, no focus dependency.
    - Remove the old `focusNiriWindow` + `setNiriColumnWidth` pattern.

    **Step 2d — Apply focus (unchanged):**
    - If `focusWindowId !== null`, call `focusNiriWindow(focusWindowId)`.
    - Then handle workspace-level focus switch-back (unchanged).

    **Import changes:**
    - Add imports: `moveColumnToIndex`, `setWindowWidth`, `listNiriWindows` (for querying window positions — actually not needed since we track IDs ourselves during Phase 1).
    - `setNiriColumnWidth` import can be REMOVED — replaced by `setWindowWidth`.
