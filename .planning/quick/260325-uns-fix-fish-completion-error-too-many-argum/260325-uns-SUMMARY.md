# Quick Task 260325-uns: Fix Fish Completion "Too Many Arguments" Error

**One-liner:** Single-quote escaping via `shellEscapeSingleQuote` applied to all 9 description interpolation sites in fish and zsh completion generators, fixing parse errors caused by apostrophes in command descriptions like "workspace's".

## What Was Built

### Root cause
Command descriptions containing apostrophes (e.g., `"Attach to a workspace's tmux session"`, `"arranges workspace windows onto a named niri workspace"`) were interpolated raw into single-quoted shell strings in the fish and zsh completion generators. The apostrophe terminated the single-quoted string early, causing fish to report "too many arguments" at the broken line.

### Fix: `shellEscapeSingleQuote` helper

Added to `src/lib/completion-generator.ts` before `buildNode`:

```typescript
/** Escape single quotes for embedding in single-quoted shell strings: ' -> '\'' */
function shellEscapeSingleQuote(s: string): string {
  return s.replace(/'/g, "'\\''")
}
```

Applied to all description interpolation sites inside single-quoted strings:

**Fish generator (4 sites):**
- `generateFish`: top-level `-d '${node.description}'` line
- `generateFish`: per-command flags `-d '${opt.description}'` line
- `emitFishSubcommands`: subcommand list `-d '${sub.description}'` line
- `emitFishSubcommands`: subcommand flags `-d '${opt.description}'` line

**Zsh generator (3 sites in `zshOptionSpec` + 2 in describe arrays):**
- `zshOptionSpec`: all 3 branches (enum, flagDynamic, plain) for `opt.description`
- `generateZshSubcmdHelperRecursive`: `sub.description` in `_describe` arrays
- `generateZsh`: `node.description` in top-level `_top_commands` array

**Bash generator: no changes** — bash uses `compgen -W "..."` and double-quoted strings where apostrophes are safe.
