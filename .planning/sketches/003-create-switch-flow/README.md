# Sketch 003: Create Switch Flow

## Design Question

How should creating new workspaces and switching among active ones fit into the same control-center TUI?

## How to View

open .planning/sketches/003-create-switch-flow/index.html

## Variants

- **A: Command Palette Create** - A lazygit-style command palette overlays the dashboard for fast create/open/sync actions.
- **B: Persistent Create Rail** - Templates and recent repos remain visible as a right rail for quick workspace creation.
- **C: Workspace Switcher Overlay** - A switcher prioritizes active state, messages, runtime, and recency for moving among environments.

## What to Look For

Compare whether creation feels like a first-class workflow without taking too much space from day-to-day management.
