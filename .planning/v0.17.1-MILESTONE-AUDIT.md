## Issues Found During Milestone Hardening

The phases did find real issues and coverage trust gaps, and those were fixed before this audit:

- `clean --gone --dry-run` entered the confirmation/removal path instead of staying preview-only.
- `workspace run --json` was accepted without `--parallel` and silently ignored the JSON contract.
- Hook execution relied on `sh` lookup instead of the explicit shell path used by the execution environment.
- Coverage reporting initially undercounted source exercised by subprocess and TUI test paths.
- Several real-source integration contracts were being tested through source-copying or over-mocked patterns instead of exercising the implementation modules.

## Accepted Gaps

These are accepted for v0.17.1 because exhaustive coverage here would add brittleness without materially improving release confidence:

- Representative command error matrices rather than every cross-product of malformed input and option combinations.
- Stable completion invariants rather than full shell completion snapshots.
- Local verification gates and readiness sentinels rather than numeric global coverage thresholds.

## Deferred External-Environment Coverage

These remain explicitly out of scope for v0.17.1 and should not block archive:

- Broad TUI rendering and dashboard rollback visibility beyond existing headless component/integration coverage.
- Live GitHub, GitLab, Gitea, Jira, and hosted forge behavior with real credentials/services.
- Real browser, window-manager, IDE, tmux, cmux, Niri, and AeroSpace launches.

## Conclusion

All current in-scope functionality is verified to the milestone's intended standard. The remaining gaps are either accepted as stable-test tradeoffs or deferred because they require live external environments or broader TUI harness work.
