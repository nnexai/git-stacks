# Milestone v0.16.0: Core Engine & Observability

**Status:** ✅ SHIPPED 2026-04-05
**Phases:** 69-73
**Total Plans:** 9

## Overview

v0.16.0 broke the oversized `workspace-ops.ts` engine into domain-cohesive modules without changing the public CLI surface, added stderr-only debug observability via `GIT_STACKS_DEBUG=1`, and locked the new boundaries down with focused extracted-module tests plus a repo-native circular-dependency gate.

## Milestone Summary

**Key Decisions:**

- Keep `workspace-ops.ts` as a stable facade during extraction, then shrink it only after callers and tests move to the new modules.
- Bootstrap observability once in `src/index.ts`, keep debug output on `stderr`, and explicitly silence logging before `manage` starts the TUI.
- Enforce the new boundaries with direct extracted-module tests and a repo-native `madge` gate instead of ad hoc verification.

**Issues Resolved:**

- Split a 1,700+ line monolith into focused env, lifecycle, status, git, and YAML modules.
- Added timing/debug observability without corrupting human-readable stdout or `--json` output.
- Removed live dashboard IPC cycles and made dependency-graph checks repeatable with `bun run test:deps`.

**Issues Deferred:**

- Broader dependency injection across every shell/process/filesystem boundary remains future work.
- Structured logging beyond the current stderr debug channel remains future work.
- Plugin-boundary formalization, rollback orchestration, and indexed config caching remain future-milestone candidates.

**Technical Debt Incurred:**

- No blocking technical debt was introduced. The only residual note is that the debug-enabled `manage` path is backed by explicit silence + smoke evidence rather than a separately recorded interactive visual artifact.
