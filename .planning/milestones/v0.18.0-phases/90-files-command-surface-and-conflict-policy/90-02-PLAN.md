<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workspace target tree -> source tree | `files push` can write private workspace changes back to source paths. |
| Source tree -> workspace target tree | `files pull` can overwrite or delete workspace-local files under force mode. |
| Dry-run plan -> user decision | The user relies on planned writes/deletes/refusals before destructive operation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-90-04 | Tampering | Default pull/push | mitigate | Default mode refuses differing paths and destination-only deletes. |
| T-90-05 | Tampering | Force mirror | mitigate | Force mode remains inside validated source/target roots and is explicitly requested. |
| T-90-06 | Repudiation | Dry-run output | mitigate | Dry-run returns planned writes, overwrites, deletes, and refusals before mutation. |
| T-90-07 | Denial of Service | Recursive replacement | mitigate | Tests cover directory replacement and containment before deletion. |
</threat_model>
