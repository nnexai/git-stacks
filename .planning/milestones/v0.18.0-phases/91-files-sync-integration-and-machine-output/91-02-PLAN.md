## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem comparison -> automation JSON | Future automation relies on fields and exit codes to distinguish drift, refusal, and error. |
| Large path lists -> stdout | Verbose details can leak or overload automation if unbounded. |
| Human output path -> JSON output path | Mixed progress text can corrupt machine parsing. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-91-05 | Denial of Service | Verbose JSON detail | mitigate | Cap details and return truncation metadata. |
| T-91-06 | Tampering | Automation exit codes | mitigate | Nonzero on applied refusals/errors; zero for status and dry-run previews. |
</threat_model>
