<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Workspace config -> create-time filesystem writes | User-controlled sync entries decide local source and target paths during workspace creation. |
| Existing workspace files -> normal open | User-edited target files must not be overwritten by opening an existing workspace. |
| Missing worktree recreation -> file materialization | Recreated worktrees can receive newly materialized files, but existing targets must remain untouched. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-91-01 | Tampering | Workspace create | mitigate | Treat create-time sync as part of create success and fail clearly on unsafe/conflicting targets. |
| T-91-02 | Tampering | Normal open | mitigate | Add tests proving normal open does not refresh existing sync targets. |
| T-91-03 | Repudiation | Missing-worktree recreation | mitigate | Restrict automatic materialization to missing targets and report progress/errors explicitly. |
</threat_model>
