## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| release smoke -> shipped docs | The release gate must detect drift between the actual CLI surface and what the docs claim. |
| verified commit -> rc tag/package | The publishable RC tag must only be created after the targeted gate and full local verify pass. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-94-05 | Tampering | release smoke coverage | mitigate | Add a dedicated RC smoke test that checks README, CHANGELOG, command help, file-sync behavior, and forge-source dry-run together. |
| T-94-06 | Repudiation | rc handoff script | mitigate | Script the exact `verify`, `bun publish --dry-run`, and tag sequence so the RC state is reproducible and attributable to one verified commit. |
| T-94-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs occur in this plan. |
</threat_model>
