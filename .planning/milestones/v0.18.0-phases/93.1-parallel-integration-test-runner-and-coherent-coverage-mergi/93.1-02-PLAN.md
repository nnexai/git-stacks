## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test fixture constants -> shared `/tmp` namespace | Hard-coded workspace/editor roots can collide across parallel integration processes. |
| Integration tests -> local git/config homes | Reused config or git fixture roots can corrupt unrelated workers or produce flakes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-93-1-04 | Tampering | Real-write integration fixtures | mitigate | Replace shared `/tmp/test-workspaces` and `/tmp/test.yml` roots with unique `makeTmpDir()`-backed paths. |
| T-93-1-05 | Repudiation | Serial-only exceptions | mitigate | Enforce named file + reason metadata through `serialOnlyFiles` and the fixture-safety audit test. |
| T-93-1-06 | Availability | Git/config fixture setup | mitigate | Preserve the repo's existing per-file temp/config/git pattern and add regression tests for path collisions before enabling wider parallel execution. |
</threat_model>
