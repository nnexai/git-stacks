# Plan 61-01: Secret Resolver Library

## Objective

Create `src/lib/secrets.ts` — the core secret resolution library with `SecretResolver` interface, `parseSecretRef()` parser, `resolveSecrets()` orchestrator, and three built-in resolvers (`keychain`, `env`, `cmd`). This is a standalone library with no dependency on workspace types.

<action>
Create `src/lib/secrets.ts` with:

1. Export the `SecretResolver` interface:
```ts
export interface SecretResolver {
  /** Matches the prefix before `:` in the reference */
  id: string
  /** Resolve the secret value from the reference path. Throws on failure. */
  resolve(path: string): Promise<string>
}
```

2. Export the reference pattern constant:
```ts
export const REF_PATTERN = /^\$\{\{\s*(\w+):(.+?)\s*\}\}$/
```

3. Export `parseSecretRef`:
```ts
export function parseSecretRef(value: string): { id: string; path: string } | null {
  const m = value.match(REF_PATTERN)
  if (!m) return null
  return { id: m[1], path: m[2].trim() }
}
```

Use relative imports only (no `@/` alias — production code).
</action>

### Task 3: Create keychain resolver with platform detection

<read_first>
- src/lib/secrets.ts (interface and env resolver from Tasks 1-2)
- src/lib/git.ts (subprocess patterns)
</read_first>

<action>
Add `keychainResolver` to `src/lib/secrets.ts`. The resolver must:

1. Parse the path as `service/account` (split on first `/`):
```ts
function parseKeychainPath(path: string): { service: string; account: string } {
  const slashIdx = path.indexOf("/")
  if (slashIdx === -1) throw new Error(`Invalid keychain path '${path}' — expected 'service/account'`)
  return { service: path.slice(0, slashIdx), account: path.slice(slashIdx + 1) }
}
```

2. Auto-detect platform and build command:
   - `process.platform === "darwin"`: `["security", "find-generic-password", "-s", service, "-a", account, "-w"]`
   - `process.platform === "linux"`: `["secret-tool", "lookup", "service", service, "account", account]`
   - Other: throw `"keychain resolver not supported on ${process.platform}"`

3. Execute with 10-second timeout using `execWithTimeout` (see Task 5).

4. Return trimmed stdout on success. On non-zero exit: throw with the command and stderr.

```ts
export const keychainResolver: SecretResolver = {
  id: "keychain",
  async resolve(path: string): Promise<string> {
    const { service, account } = parseKeychainPath(path)
    const cmd = process.platform === "darwin"
      ? ["security", "find-generic-password", "-s", service, "-a", account, "-w"]
      : process.platform === "linux"
      ? ["secret-tool", "lookup", "service", service, "account", account]
      : null
    if (!cmd) throw new Error(`keychain resolver not supported on ${process.platform}`)
    const result = await execWithTimeout(cmd, 10_000)
    if (!result.ok) {
      throw new Error(`Failed to resolve keychain:${path}: ${result.error}`)
    }
    return result.stdout.trim()
  },
}
```
</action>

<acceptance_criteria>
- `keychainResolver.id` equals `"keychain"`
- `parseKeychainPath("my-service/my-account")` returns `{ service: "my-service", account: "my-account" }`
- `parseKeychainPath("no-slash")` throws error containing `"expected 'service/account'"`
- On `process.platform === "darwin"`, resolver constructs command with `security find-generic-password`
- On `process.platform === "linux"`, resolver constructs command with `secret-tool lookup`
</acceptance_criteria>

### Task 5: Create execWithTimeout utility

<read_first>
- src/lib/secrets.ts (all resolver code that needs this)
- src/lib/lifecycle.ts (_exec pattern for subprocess execution)
</read_first>

<action>
Add `execWithTimeout` as a private helper in `src/lib/secrets.ts` (not exported — internal to resolvers):

```ts
async function execWithTimeout(
  cmd: string[],
  timeoutMs: number
): Promise<{ ok: boolean; stdout: string; error?: string }> {
  const proc = Bun.spawn(cmd, { stdout: "pipe", stderr: "pipe" })
  let timedOut = false
  const timer = setTimeout(() => {
    timedOut = true
    proc.kill()
  }, timeoutMs)
  
  try {
    const exitCode = await proc.exited
    clearTimeout(timer)
    
    if (timedOut) {
      return { ok: false, stdout: "", error: `timeout after ${timeoutMs / 1000}s` }
    }
    
    const stdout = await new Response(proc.stdout).text()
    
    if (exitCode !== 0) {
      const stderr = await new Response(proc.stderr).text()
      return { ok: false, stdout: "", error: `exit ${exitCode}: ${stderr.trim()}` }
    }
    
    return { ok: true, stdout }
  } catch (err) {
    clearTimeout(timer)
    if (timedOut) {
      return { ok: false, stdout: "", error: `timeout after ${timeoutMs / 1000}s` }
    }
    return { ok: false, stdout: "", error: String(err) }
  }
}
```

Note: Read stdout/stderr BEFORE calling `proc.exited` to avoid deadlock on large outputs, or use `Bun.readableStreamToText`. Adjust ordering if Bun's API requires it — the key invariant is that timeout kills the process and returns a clear error.
</action>

<acceptance_criteria>
- `execWithTimeout(["echo", "test"], 5000)` returns `{ ok: true, stdout: "test\n" }`
- `execWithTimeout(["false"], 5000)` returns `{ ok: false, stdout: "", error: ... }` with non-zero exit
- `execWithTimeout(["sleep", "30"], 100)` returns `{ ok: false, ..., error: ... }` containing `"timeout"`
</acceptance_criteria>

### Task 6: Create resolveSecrets orchestrator

<read_first>
- src/lib/secrets.ts (all previous tasks)
- .planning/phases/61-secrets/61-CONTEXT.md (D-11: signature constraint)
</read_first>

<action>
Add `resolveSecrets` to `src/lib/secrets.ts`:

```ts
export async function resolveSecrets(
  env: Record<string, string>,
  resolvers: SecretResolver[]
): Promise<Record<string, string>> {
  const entries = await Promise.all(
    Object.entries(env).map(async ([key, value]) => {
      const ref = parseSecretRef(value)
      if (!ref) return [key, value] as const
      
      const resolver = resolvers.find(r => r.id === ref.id)
      if (!resolver) {
        throw new Error(
          `No resolver for '${ref.id}' (referenced by ${key}). ` +
          `Available resolvers: ${resolvers.map(r => r.id).join(", ") || "none"}. ` +
          `Enable it in config.yml secrets.resolvers.`
        )
      }
      
      return [key, await resolver.resolve(ref.path)] as const
    })
  )
  return Object.fromEntries(entries)
}
```

Key properties:
- Resolves all references in parallel (`Promise.all`)
- Plain values pass through unchanged
- Missing resolver → throws with available resolver list
- Resolver failure → propagates the resolver's error
- Signature is `(Record<string, string>, SecretResolver[]) → Record<string, string>` — never accepts a Workspace object (D-11)
</action>

<acceptance_criteria>
- `resolveSecrets({ A: "plain", B: "${{ env:HOME }}" }, [envResolver])` returns object where `A` equals `"plain"` and `B` equals `process.env.HOME`
- `resolveSecrets({ A: "${{ unknown:x }}" }, [envResolver])` rejects with error containing `"No resolver for 'unknown'"`
- `resolveSecrets({}, [])` returns `{}`
- `resolveSecrets({ A: "no-ref", B: "also-plain" }, [])` returns `{ A: "no-ref", B: "also-plain" }`
</acceptance_criteria>
