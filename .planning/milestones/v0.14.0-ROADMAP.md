# Milestone v0.14.0: Workflow Completion & Workspace UX

**Status:** ✅ SHIPPED 2026-04-03
**Phases:** 58-63
**Total Plans:** 18

## Overview

v0.14.0 completed the core multi-repo workspace workflow: users can now see ahead/behind drift, push workspace branches in parallel, organize workspaces with labels, resolve secrets at runtime, and safely sync dirty worktrees with stash support. The milestone also closed the loop with release metadata and docs for the shipped behavior.

## Phases

### Phase 58: Ahead/Behind Tracking

**Goal:** Users can see how far each workspace's branches are from their base branch, with staleness awareness.
**Depends on:** Phase 57
**Plans:** 4 plans

Highlights:

- Added `getCommitsAhead()` and worktree-safe `isFetchStale()` primitives in `src/lib/git.ts`
- Aggregated ahead/behind into workspace status and list info in `workspace-ops.ts`
- Surfaced `↑N` / `↓N` state in `git-stacks list`, `git-stacks status`, and dashboard row/detail views
- Completed AB-01 through AB-07 with verification coverage

### Phase 59: Push

**Goal:** Users can push workspace branches to remote from a single command, with parallel execution and TUI support.
**Depends on:** Phase 58
**Plans:** 4 plans

Highlights:

- Added `pushBranch()` with force/lease/upstream handling and normalized failure reasons
- Added `pushWorkspace()` with parallel worktree pushes, trunk skips, and progress rows
- Shipped `git-stacks push [workspace]` with `--force`, `--force-with-lease`, `--dry-run`, `--set-upstream`, and `--json`
- Added dashboard push action and progress dialog

### Phase 60: Labels

## Milestone Summary

**Key accomplishments:**

1. Added ahead/behind tracking across git primitives, CLI list/status, and dashboard row/detail surfaces.
2. Added a workspace-wide push flow with parallel repo execution and matching dashboard progress UI.
3. Added label-based organization across config, CLI CRUD/filter/create, and dashboard grouping/filtering.
4. Added runtime-only secret resolution for open-time env injection and matching env preview behavior.
5. Added stash-on-sync so dirty worktrees can be synced safely without manual stash juggling.

**Key decisions:**

- Ahead/behind staleness uses `git rev-parse --git-common-dir` so worktree FETCH_HEAD detection stays correct.
- Ahead aggregates as sum, behind as max, and stale as OR across worktree repos.
- Secret refs remain persisted as `${{ resolver:path }}` and resolve only in runtime env maps.
- Auto-stash uses a fixed `git-stacks auto-stash (sync)` marker so double-stash detection is reliable.

**Issues resolved:**

- Closed the workflow gap between `open` and `env` by routing both through the same secret-resolution pipeline.
- Closed release-doc coverage gaps for ahead/behind visibility and env preview semantics before archival.

**Issues deferred:**

- Dashboard label-edit inline input is present in state/types but does not yet have a menu/key entry point.
- Grouped label headers remain color-only because current OpenTUI text props do not support bold styling.

**Technical debt incurred:**

- None beyond the non-blocking dashboard label UI follow-ups listed above.
