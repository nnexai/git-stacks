# Phase 32 Plan 01: GitLab Branch Slash Root Cause Documentation

## Goal
Document the root cause of GitLab branch name '/' failures and add a glab version check to `git-stacks doctor`.

## Investigation Findings (Pre-Plan)

**Root cause: glab CLI bug, not our code.**

- **glab issue [#948](https://gitlab.com/gitlab-org/cli/-/issues/948)**: `glab repo view --web` fails to URL-encode branch names containing special characters (`/`, `#`, etc.) when constructing the browser URL
- **glab MR [!1183](https://gitlab.com/gitlab-org/cli/-/merge_requests/1183)**: Fixed by calling `url.PathEscape` on branch names before URL construction. Merged Feb 2023.
- **Our code** (`src/lib/integrations/gitlab.ts`): Does not manipulate branch names — just passes `["repo", "view", "--web"]` to `glab` via `Bun.spawn`. The bug is entirely in glab's URL construction.

**Affected command**: Only `git-stacks integration gitlab open --web` (which calls `glab repo view --web`). MR commands (`mr create`, `mr view`) are unaffected because they use API calls, not URL construction.

## Tasks
