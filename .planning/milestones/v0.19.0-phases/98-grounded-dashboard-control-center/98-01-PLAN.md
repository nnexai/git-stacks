<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workspace config -> grouping labels | User-authored labels/templates influence grouping headers rendered in the terminal. |
| workspace status -> state grouping | Existing loaded/loading/error status drives scan-focused state groups. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-98-01 | Spoofing | grouping headers | mitigate | Derive headers from explicit `WorkspaceGroupingMode` rules and keep row status tokens visible so headers are not the only source of truth. |
| T-98-02 | Denial of Service | grouped list rendering | mitigate | Keep grouping as an in-memory transformation of already-loaded entries and preserve viewport slicing. |
| T-98-SC | Tampering | dependencies | mitigate | No package installs or new UI libraries are allowed in this plan. |
</threat_model>
