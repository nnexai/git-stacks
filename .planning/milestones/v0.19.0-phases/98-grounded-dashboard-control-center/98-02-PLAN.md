## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| notes storage -> TUI detail | Workspace note files are user-authored config-root data rendered in the dashboard. |
| file-status helper -> TUI detail | Filesystem-derived sync state enters the TUI through the Phase 97 grouped model. |
| keyboard input -> scroll state | Detail-scroll keys must not corrupt list selection or action-menu state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-98-03 | Information Disclosure | notes detail | mitigate | Render only workspace-scoped notes from `src/lib/notes.ts` for the selected workspace; do not scan other config directories. |
| T-98-04 | Tampering | file-status rendering | mitigate | Consume Phase 97 file-status model directly and forbid CLI subprocess or duplicated sync policy in production dashboard code. |
| T-98-05 | Denial of Service | long detail content | mitigate | Render a bounded viewport with scroll offset instead of allowing unbounded detail rows to overflow the screen. |
| T-98-SC | Tampering | dependencies | mitigate | No package installs or external UI components are allowed in this plan. |
</threat_model>
