<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TUI menu -> registry editor | Keyboard action launches an external editor for user-managed registry YAML. |
| editor result -> repo list reload | Edited YAML is validated before the dashboard resumes normal rendering. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99-01 | Tampering | `registry.yml` edit flow | mitigate | Use `editRegistryYaml()` validation instead of a custom parser or unchecked write path. |
| T-99-02 | Denial of Service | renderer suspend/resume | mitigate | Wrap editor launch in `try/finally` so the renderer resumes and repo list reloads after editor exit or failure. |
| T-99-SC | Tampering | dependencies | mitigate | No package-manager changes; use existing editor and OpenTUI test infrastructure. |
</threat_model>
