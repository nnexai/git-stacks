## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Template YAML -> created workspace YAML | Template-defined command strings are copied into persisted workspace config. |
| Source workspace YAML -> cloned workspace YAML | Clone must preserve copied commands without accidentally re-resolving or dropping them. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-04 | Tampering | `src/tui/workspace-wizard.ts` / `App.tsx` snapshot wiring | mitigate | Reuse the existing template snapshot pattern so manual commands persist through the same reviewed path as hooks/env/files. |
| T-95-05 | Repudiation | `src/tui/workspace-clone.ts` clone persistence | mitigate | Clone preserves copied command maps from the source workspace rather than re-reading templates, keeping saved state explicit and auditable. |
| T-95-06 | Elevation of Privilege | Template-backed dashboard create path | accept | The feature intentionally copies user-authored shell strings; Phase 95 keeps them explicit and manually invoked rather than auto-running during creation. |
</threat_model>
