<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI input -> workspace detection | Omitted workspace names rely on cwd detection and can target the wrong workspace if detection is ambiguous. |
| CLI `--skip-secrets` and `run` flags -> execution engine | User-facing flags alter secret resolution and execution behavior for shell commands. |
| Command tree -> verification inventory | Adding a new command family without inventory updates causes local verification drift. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-11 | Spoofing | `src/commands/command.ts` workspace resolution | mitigate | Reuse explicit-workspace-first, cwd-fallback-second behavior from `files`, and fail with a clear usage error when detection is ambiguous or absent. |
| T-95-12 | Information Disclosure | `--skip-secrets` CLI contract | mitigate | Expose `--skip-secrets` explicitly in the command surface so users can avoid resolving secrets for inspection or lower-trust runs. |
| T-95-13 | Repudiation | CLI exit status and dry-run output | mitigate | Print the exact dry-run order and propagate the real exit status from the execution engine so failures are attributable to a concrete command entry. |
| T-95-14 | Tampering | `tests/e2e-inventory.ts` drift | mitigate | Update the canonical inventory and verify with `bun run verify:gates` so the shipped command surface cannot silently outrun coverage metadata. |
</threat_model>
