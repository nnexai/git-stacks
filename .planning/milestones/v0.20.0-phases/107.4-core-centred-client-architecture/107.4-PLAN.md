# Phase 107.4 Plan: Core-Centred Client Architecture

## Objective

Remove the OpenTUI dashboard as a second machine-side application. Make the local TypeScript service the sole interactive authority while preserving the browser's narrower security projection and each client's native rendering model.

## Implementation

1. Define a complete trusted core projection and typed official service client.
2. Expand the operation registry to all dashboard lifecycle, Git, command, issue, label, template, and repository mutations.
3. Move OpenTUI reads, status, notes, signals, creation, and mutations to the service client and event stream.
4. Share browser/OpenTUI transport types, signal presentation, priority ordering, and configured-command execution semantics.
5. Isolate foreground `$EDITOR` and attached-shell behavior as explicit client handoffs.
6. Add transport, conformance, and architecture-boundary tests; run complete release gates.
