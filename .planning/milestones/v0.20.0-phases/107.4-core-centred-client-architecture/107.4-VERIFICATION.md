# Phase 107.4 Verification

## Outcome

Verified. The Bun service is the sole machine-side authority for both interactive clients. The OpenTUI dashboard is a service-backed renderer, while the browser continues to consume its narrower browser-safe projection.

## Requirement evidence

| Requirement | Evidence |
|---|---|
| CORE-01 | `core-contract.ts`, `core-state.ts`, and `GET /v1/core` define and serve the complete trusted projection. |
| CORE-02 | `operations.ts` adapts every dashboard lifecycle, Git, command, issue, label, template, and repository mutation; `App.tsx` submits those intents through `service/client.ts`. |

## Resilience and resource checks

- The dashboard maintains one long-lived service event stream instead of one-second signal/status polling.
- Trusted clients reject dead discovery descriptors and bootstrap the shared service in a detached process, so closing the TUI does not keep the command alive or terminate browser-owned state.
- Dashboard teardown aborts its long-lived SSE stream; detached service startup strips inherited workspace, repository, surface, and signal-session context.
- API SSE streams flush an immediate readiness marker, removing the idle-heartbeat delay before operation submission.
- Workspace files and notes are cached per workspace; file status is invalidated by the authoritative core revision instead of cursor movement.
- Signal SSE events update the local projection directly rather than issuing a replacement projection request for every event.
- Short TUI lists collapse to their content height and detail overflow uses OpenTUI's measured scroll container; no service-side pagination or viewport shaping is used for ordinary detail data.
- Notification and activity dismissals are durable shared-state mutations; both clients filter dismissed IDs, and the TUI removes them optimistically.
- Snapshot invalidations compare revisions and wait for an in-flight foreground refresh, preventing duplicate full projections.
- Interactive operations use SSE for immediate progress and fall back to the durable operation resource if the stream disconnects.
- Cancellation before SSE readiness is bounded and covered by an isolated regression test.
- Configured-command output is journaled in batches of up to 16 lines or 50 ms rather than one durable write per line.
- The dependency scan processed 149 source files with no circular dependency.

## Deliberate boundary

`$EDITOR` and an attached foreground workspace shell still require the TUI to yield its controlling terminal. The service resolves the target and workspace context; the renderer owns only the terminal attachment. These are presentation handoffs, not alternate domain implementations.
