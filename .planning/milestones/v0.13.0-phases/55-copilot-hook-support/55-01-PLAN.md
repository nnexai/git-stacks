### Copilot JSON types (internal to this file)

Define internal types for the Copilot hooks JSON format:

```typescript
interface CopilotHookEntry {
  type: "command"
  bash: string
  env?: Record<string, string>
  timeoutSec?: number
  comment?: string
}

interface CopilotHooksFile {
  version: 1
  hooks: Record<string, CopilotHookEntry[]>
}
```

### Helper functions

1. **`getHooksPath(repoWorktreePath: string): string`** — returns `join(repoWorktreePath, ".github", "hooks", "git-stacks.json")`

2. **`entriesToCopilotConfig(entries: HookEntry[], workspaceName: string): CopilotHooksFile`**
   - Create an empty `CopilotHooksFile` with `version: 1` and empty `hooks` object
   - Map from `HookEntry` event names (Claude-style) to Copilot event names:
     - `"Stop"` → `"sessionEnd"`
     - `"PreToolUse"` → `"preToolUse"`
     - `"PostToolUse"` → `"postToolUse"`
     - `"UserPromptSubmit"` → `"userPromptSubmitted"`
   - For each entry, create a `CopilotHookEntry` with:
     - `type: "command"`
     - `bash`: the entry's `command` field value (inline bash, not a file path — per decision D-05)
     - `env`: `{ GS_WORKSPACE_NAME: workspaceName, GS_FROM: "copilot" }` (per decision D-10)
     - `timeoutSec: 10` (notification hooks should be fast)
     - `comment`: a descriptive string like `"git-stacks: ${entry.event}${entry.matcher ? ` (${entry.matcher})` : ''} notification"`
   - For entries with a `matcher` (e.g. `PreToolUse` with `AskUserQuestion`), wrap the bash command in a stdin-filtering script:
     ```
     input=$(cat); toolName=$(echo "$input" | grep -o '"toolName":"[^"]*"' | head -1 | cut -d'"' -f4); if [ "$toolName" = "AskUserQuestion" ]; then ${entry.command}; fi
     ```
     This parses stdin JSON to check `toolName` before executing the notification command (per decision D-09). Use lightweight grep/cut instead of requiring jq.
   - Group entries by mapped Copilot event name into the `hooks` object arrays

### Plugin export

```typescript
export const copilotPlugin: AgentHookPlugin = {
  id: "copilot",
  label: "GitHub Copilot",

  generateHookEntries(workspaceName: string): HookEntry[] { ... },
  install(repoWorktreePath: string, workspaceName: string): void { ... },
  remove(repoWorktreePath: string): void { ... },
}
```

**`generateHookEntries(workspaceName)`** — Return the same 4 notification HookEntry objects as claude-code (same commands, same events), with workspace name interpolated:
- `Stop` → `git-stacks message send "Copilot has finished and may need your attention" --workspace ${workspaceName} --from copilot`
- `PreToolUse` + matcher `AskUserQuestion` → `git-stacks message send "Copilot is asking a question — input needed" --workspace ${workspaceName} --from copilot`
- `UserPromptSubmit` → `git-stacks message clear --workspace ${workspaceName} --from copilot`
- `PostToolUse` + matcher `AskUserQuestion` → `git-stacks message clear --workspace ${workspaceName} --from copilot`

**`install(repoWorktreePath, workspaceName)`**:
1. Generate entries via `this.generateHookEntries(workspaceName)`
2. Convert to Copilot JSON via `entriesToCopilotConfig(entries, workspaceName)`
3. Ensure directory exists: `mkdirSync(join(repoWorktreePath, ".github", "hooks"), { recursive: true })`
4. Write JSON: `writeFileSync(hooksPath, JSON.stringify(config, null, 2) + "\n")`
5. Idempotency is automatic — overwrite the entire file each time (per decision D-12, Copilot plugin owns the file)

**`remove(repoWorktreePath)`**:
1. Get hooks path via `getHooksPath(repoWorktreePath)`
2. If file exists, delete it: `rmSync(hooksPath, { force: true })`
3. Optionally remove empty `.github/hooks/` directory (if empty after deletion). Use `readdirSync` to check; if empty, `rmdirSync`. Do NOT remove `.github/` itself — other files may live there.
4. No-op if file doesn't exist

Use relative imports throughout. Follow kebab-case filename convention.
