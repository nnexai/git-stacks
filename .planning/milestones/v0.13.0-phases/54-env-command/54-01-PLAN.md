# Plan 54-01: Env Formatting Functions and Repo Detection Helper

<objective>
Create the core library for the `git-stacks env` command: formatting functions that convert env var records into table, shell, dotenv, and JSON output, plus a helper to detect which repo the CWD is inside.
</objective>

## Tasks

<task id="54-01-01">
<title>Create env formatting library</title>
<read_first>
- src/lib/workspace-ops.ts (lines 108-146 — mergeEnv, buildBaseEnv, buildRepoEnv signatures and implementation)
- src/lib/paths.ts (getTasksDir function)
- src/lib/config.ts (readWorkspace, readGlobalConfig, listWorkspaces types)
</read_first>
<action>
Create `src/lib/env.ts` with the following exports:

1. **`type EnvFormat = "table" | "shell" | "dotenv" | "json"`**

2. **`function formatEnvTable(env: Record<string, string>): string`**
   - Sort keys alphabetically
   - Compute max key length for alignment
   - Output lines: `KEY_NAME  value` with 2-space gap after padded key column
   - Return the full formatted string (no trailing newline — caller adds)

3. **`function formatEnvShell(env: Record<string, string>): string`**
   - Sort keys alphabetically
   - Output lines: `export KEY=value` per entry
   - Quote values with double quotes when they contain: space, `"`, `'`, `$`, backtick, `#`, newline, tab, `;`, `(`, `)`, `|`, `&`, `<`, `>`
   - Inside double-quoted values, escape `"` as `\"` and `$` as `\$` and backtick as `` \` ``
   - Unquoted values are bare

4. **`function formatEnvDotenv(env: Record<string, string>): string`**
   - Sort keys alphabetically
   - Output lines: `KEY=value` per entry
   - Quote values with double quotes when they contain: space, `"`, `'`, `#`, newline, tab
   - Inside double-quoted values, escape `"` as `\"` and newline as `\n`
   - Unquoted values are bare

5. **`function formatEnvJson(env: Record<string, string>): string`**
   - Return `JSON.stringify(env, Object.keys(env).sort(), 2)`

6. **`function formatEnv(env: Record<string, string>, format: EnvFormat): string`**
   - Dispatch to the appropriate formatter based on `format`

7. **`function needsShellQuoting(value: string): boolean`** (internal helper, not exported)
   - Returns true if value contains any of: space, `"`, `'`, `$`, backtick, `#`, newline, tab, `;`, `(`, `)`, `|`, `&`, `<`, `>`

8. **`function needsDotenvQuoting(value: string): boolean`** (internal helper, not exported)
   - Returns true if value contains any of: space, `"`, `'`, `#`, newline, tab

Use relative imports only (no `@/` alias). Follow kebab-case filename convention.
</action>
<acceptance_criteria>
- `src/lib/env.ts` exists and exports `EnvFormat`, `formatEnvTable`, `formatEnvShell`, `formatEnvDotenv`, `formatEnvJson`, `formatEnv`

- `formatEnvShell({ KEY: "hello world" })` outputs `export KEY="hello world"`
- `formatEnvShell({ KEY: "simple" })` outputs `export KEY=simple` (no quotes for safe values)

</acceptance_criteria>
</task>

<task id="54-01-02">
<title>Create repo detection helper</title>
<read_first>
- src/lib/workspace-ops.ts (lines 1344-1384 — detectWorkspaceFromCwd and CwdDetectionResult)
- src/lib/config.ts (Workspace type, WorkspaceRepo shape)
- src/lib/paths.ts (expandHome)
</read_first>
<action>
Add to `src/lib/env.ts`:

1. **`function detectRepoFromCwd(workspace: Workspace, cwd?: string): string | null`**
   - Takes a known workspace and an optional CWD (defaults to `process.cwd()`)
   - Iterates `workspace.repos` where `mode === "worktree"`
   - For each repo, resolve `expandHome(repo.task_path)` using `resolve()` from "path"
   - If CWD === resolvedTaskPath or CWD starts with resolvedTaskPath + "/", track the match (longest path wins, same as `detectWorkspaceFromCwd` logic)
   - Return the matched repo's `name` or `null` if no match

Import `Workspace` type from `../lib/config` (type-only import). Import `expandHome` from `../lib/paths`. Import `resolve` from `path`.
</action>
<acceptance_criteria>
- `detectRepoFromCwd` is exported from `src/lib/env.ts`
- Given workspace with repo `{ name: "api", mode: "worktree", task_path: "/tmp/tasks/ws/api" }`, calling `detectRepoFromCwd(ws, "/tmp/tasks/ws/api/src")` returns `"api"`
- Given same workspace, calling `detectRepoFromCwd(ws, "/some/other/path")` returns `null`
- Repos with `mode: "trunk"` are skipped
- `bun run typecheck` passes
</acceptance_criteria>
</task>
