# Plan 56-01: Conditional Forge CLI Checks and Tmux Config Example

<objective>
Make `git-stacks doctor` only check forge CLI binaries when the respective integration is configured, and update the tmux integration `configExample` to show a practical pane layout.
</objective>

## must_haves

- Doctor only warns about missing `gh` when `github` integration is enabled in global config
- Doctor only warns about missing `glab` when `gitlab` integration is enabled in global config
- Doctor only warns about missing `tea` when `gitea` integration is enabled in global config
- Doctor only warns about missing `jira` when `jira` integration is enabled in global config
- When no forge integrations are configured, doctor produces zero forge CLI warnings
- Tmux `configExample` string includes `panes:` array with `direction`, `surfaces`, and `command` keys

2. Replace lines 317-348 (the four unconditional forge CLI checks) with a data-driven loop:
   ```ts
   // Forge CLI availability — only check when the integration is configured (DOC-01)
   const forgeClis = [
     { integrationId: "github", binary: "gh", label: "gh (GitHub CLI)", install: "https://cli.github.com/", missingMsg: "not installed — GitHub PR commands unavailable" },
     { integrationId: "gitlab", binary: "glab", label: "glab (GitLab CLI)", install: "https://gitlab.com/gitlab-org/cli", missingMsg: "not installed — GitLab MR commands unavailable" },
     { integrationId: "gitea", binary: "tea", label: "tea (Gitea CLI)", install: "https://gitea.com/gitea/tea", missingMsg: "not installed — Gitea PR commands unavailable" },
     { integrationId: "jira", binary: "jira", label: "jira (Jira CLI)", install: "https://github.com/ankitpokhrel/jira-cli", missingMsg: "not installed — Jira issue commands will use configurable template fallback" },
   ]

   for (const { integrationId, binary, label, install, missingMsg } of forgeClis) {
     if (!resolveEnabledGlobally(integrationId, false, config)) continue
     const available = await checkBinary(binary)
     binaryIssues.push({
       icon: available ? "pass" : "warn",
       entity: label,
       message: available ? "installed" : missingMsg,
       fix: available ? undefined : { action: "info", message: `Install: ${install}` },
     })
   }
   ```

   Key behavioral change: Each forge CLI check is now gated by `resolveEnabledGlobally(integrationId, false, config)`. Since all forge integrations have `enabledByDefault: false`, when no config entry exists for an integration, the check is skipped entirely. When `config.integrations.github = { enabled: true }`, the `gh` check runs.
</action>
