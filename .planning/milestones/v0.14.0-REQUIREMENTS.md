## v0.14.0 Requirements

Requirements for milestone v0.14.0: Workflow Completion & Workspace UX. Each maps to roadmap phases.

### Ahead/Behind Tracking

- [x] **AB-01**: User can see per-repo commit distance from base branch via `getCommitsAhead` primitive
- [x] **AB-02**: Workspace list info includes aggregated ahead (sum) and behind (max) across worktree repos
- [x] **AB-03**: `git-stacks list` displays AHEAD and BEHIND columns
- [x] **AB-04**: `git-stacks status <name>` shows per-repo ahead/behind counts
- [x] **AB-05**: TUI WorkspaceRow displays `â†‘N â†“N` indicators after branch name
- [x] **AB-06**: TUI WorkspaceDetail shows per-repo ahead/behind in repo table
- [x] **AB-07**: Staleness detection via FETCH_HEAD mtime with 15-minute threshold and `aheadBehindStale` flag

### Push

- [x] **PUSH-01**: `pushBranch` git primitive supports `--force`, `--force-with-lease`, and `--set-upstream` options
- [x] **PUSH-02**: `pushWorkspace` function executes push in parallel across repos, skips trunk repos, reports via progress callback
- [x] **PUSH-03**: `git-stacks push [name]` CLI command with `--force-with-lease`, `--force`, `--dry-run`, and `--json` flags
- [x] **PUSH-04**: TUI ActionMenu includes push action with live per-repo progress display

### Labels

- [x] **LBL-01**: `labels` field on WorkspaceSchema and TemplateSchema (optional string array with regex validation)
- [x] **LBL-02**: `git-stacks label add/remove/list/clear <workspace>` subcommand for direct label management
- [x] **LBL-03**: `--label` filter on `git-stacks list` with AND logic when multiple labels specified
- [x] **LBL-04**: `--label` flag on `git-stacks new` sets labels at workspace creation time
- [x] **LBL-05**: TUI WorkspaceRow renders label tags after branch name
- [x] **LBL-06**: TUI filter (`/`) matches against workspace labels in addition to workspace name
- [x] **LBL-07**: TUI group-by-label toggle (`g` key) with grouped view and [unlabeled] section
- [x] **LBL-08**: Template labels unioned onto workspace at creation time (not inherited dynamically)

### Secrets

- [x] **SEC-01**: `${{ resolver:path }}` reference syntax parsed by `parseSecretRef` function
- [x] **SEC-02**: `resolveSecrets` function resolves all secret references in an env map
- [x] **SEC-03**: `keychain` resolver auto-detects platform â€” macOS `security` CLI / Linux `secret-tool`
- [x] **SEC-04**: `env` resolver reads values from `process.env` without subprocess
- [x] **SEC-05**: `cmd` resolver executes via `sh -c` and requires explicit opt-in in global config
- [x] **SEC-06**: Secret resolution integrated in `openWorkspace` between `mergeEnv()` and `writeEnvFiles()`
- [x] **SEC-07**: `--skip-secrets` flag on open/new logs warnings and substitutes empty strings
- [x] **SEC-08**: Resolved secret values are never written back to workspace YAML
- [x] **SEC-09**: `secrets.resolvers` field in global config controls which resolvers are available
- [x] **SEC-10**: External CLI resolvers enforce 10-second subprocess timeout

### Stash

- [x] **STH-01**: `stashPush` git primitive with `--include-untracked`, custom message, and stashRef return
- [x] **STH-02**: `stashPop` git primitive detects conflicts from exit code and stderr
- [x] **STH-03**: `--stash` flag on `git-stacks sync` auto-stashes dirty repos before sync, pops in reverse after
- [x] **STH-04**: Stash pop failure preserves stash, emits recovery command, and sets `ok: false` on SyncResult
