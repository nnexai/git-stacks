# Requirements: git-stacks v0.16.0

**Defined:** 2026-04-05
**Core Value:** One command takes you from "I need to work on feature X" to a fully running dev environment.

## v0.16.0 Requirements

Requirements for v0.16.0 Core Engine & Observability.

### Module Extraction

- [x] **EXTR-01**: `workspace-ops.ts` is split into domain modules with `workspace-ops.ts` as a stable facade during extraction
- [x] **EXTR-02**: `workspace-env.ts` extracted — `mergeEnv`, `buildBaseEnv`, `buildRepoEnv`, `buildWorkspaceEnv`, `writeEnvFiles`, `resolveWorkspaceEnvVars`
- [x] **EXTR-03**: `workspace-lifecycle.ts` extracted — close/clean/remove/merge with cascade helpers (`_executeClose`, `_executeClean`) co-located
- [x] **EXTR-04**: `workspace-git.ts` extracted — sync/push/pull operations
- [x] **EXTR-05**: `workspace-status.ts` extracted — `getWorkspaceStatus`, `getDirtyWorktrees`, `getWorkspaceListInfo`, `detectWorkspaceFromCwd`
- [x] **EXTR-06**: `workspace-yaml.ts` extracted — YAML editors, rename ops, CWD-aware edit support
- [x] **EXTR-07**: Each extracted module exposes the established `_exec` seam where it owns process spawning
- [x] **EXTR-08**: All existing tests pass after each extraction step
- [x] **EXTR-09**: Cascade ordering (remove → clean → close) is preserved end-to-end

### Observability

- [x] **OBSV-01**: `GIT_STACKS_DEBUG=1` enables debug trace output to stderr
- [x] **OBSV-02**: Each domain module emits labeled debug lines: `[module] operation: detail`
- [x] **OBSV-03**: Operation timing uses `[module] step: Xms` format behind the debug flag
- [x] **OBSV-04**: Debug output is silent by default — zero user-facing noise on normal CLI invocations
- [x] **OBSV-05**: TUI-safe — debug output stays off the `manage` surface; smoke run confirms zero stderr bytes before timeout

### Testing

- [x] **TEST-01**: Unit tests exist for extracted workspace-env helpers without real git repos
- [x] **TEST-02**: Unit tests exist for extracted workspace-status query functions
- [x] **TEST-03**: Unit tests exist for extracted workspace-git operations using mocks
- [x] **TEST-04**: Circular import detection is enforced and verified with `madge`

## Out of Scope

| Feature | Reason |
|---------|--------|
| DI container/framework | `_exec` seams already fit the codebase better than a container |
| Structured JSON logging | CLI stderr debug is the right primitive for the current scope |
| OpenTelemetry tracing | Single-process CLI; too much machinery for the problem |
| Class-based domain objects | No benefit over module-level functions in this codebase |
| Caller import path migration | Direct caller migration beyond the extracted modules remains future work |
