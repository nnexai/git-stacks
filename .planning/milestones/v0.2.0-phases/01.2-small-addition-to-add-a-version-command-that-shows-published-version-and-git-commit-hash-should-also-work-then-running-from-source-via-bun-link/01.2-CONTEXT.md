# Phase 01.2: Version Command - Context

**Gathered:** 2026-03-17
**Status:** Ready for planning

<domain>
## Phase Boundary

Enhance the existing `-V`/`--version` flag so it outputs the npm package version plus the git commit hash (with dirty flag) when running from source. No new subcommand. Purely additive — no behavioral changes to any other command.

</domain>

<decisions>
## Implementation Decisions

### Command surface
- Enhance the existing `-V`/`--version` flag only — no new `version` subcommand
- The current hardcoded `"0.1.1"` in `index.ts:33` is replaced with a dynamically built string
- Commander's `.version(string)` call is what drives the output — we just pass a richer string

### Output format
- Format: `{version} ({short-hash})` when hash available
- Format: `{version} ({short-hash}-dirty)` when hash available and working tree has uncommitted changes
- Format: `{version}` (hash omitted entirely) when not in a git repo (npm global install)
- Short hash = 7-char abbreviated commit hash from `git rev-parse --short HEAD`
- Dirty detection via `git status --porcelain` — any output = dirty

### Git hash fallback
- When `git rev-parse --short HEAD` fails (no .git dir, outside a git repo), omit the hash entirely
- Published npm install shows just `0.1.3` — clean, no `(unknown)` noise
- `bun link` from source shows `0.1.3 (26d9ea8)` or `0.1.3 (26d9ea8-dirty)`

### Version source
- Read version from `package.json` at startup via `Bun.file(import.meta.dir + "/../package.json").json()`
- Avoids the current drift problem (hardcoded `"0.1.1"` while `package.json` is at `"0.1.3"`)
- Works identically with `bun link` since `import.meta.dir` resolves to the actual source directory

### Claude's Discretion
- Exact async/await structure for the version string construction
- Whether to extract into a helper function or inline in `index.ts`
- Error handling for malformed package.json (safe to assume well-formed in this project)

</decisions>

## Existing Code Insights

### Reusable Assets
- `src/index.ts:12-26` — `checkGitVersion()`: pattern for running git commands with `.quiet().nothrow()` and checking `exitCode` — use same pattern for `git rev-parse` and `git status`
- `src/lib/git.ts` — git operations via Bun `$` shell; same shell available in `index.ts`

### Established Patterns
- `$ git --version`.quiet().nothrow()` — the established pattern for safe git calls that may fail; apply to hash/dirty checks
- `import.meta.dir` — resolves to actual source dir in both `bun run src/index.ts` and `bun link` scenarios

### Integration Points
- `src/index.ts:33`: `program.name(binName).description(...).version("0.1.1")` — the single line to modify
- Version string must be built before `.version()` is called; since `Bun.file(...).json()` is async, the top-level `index.ts` (already `async` via `await checkGitVersion()`) can await it inline

</code_context>
