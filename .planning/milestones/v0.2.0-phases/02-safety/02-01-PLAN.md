    Add a new exported function `warnExternalFiles` to `src/lib/files.ts`:

    ```typescript
    export function warnExternalFiles(
      workspace: Workspace,
      stacks: Map<string, Stack>,
      wsInstanceRoot: string,
      tasksDir: string
    ): string[]
    ```

    **Implementation logic:**
    1. Collect all resolved destinations from workspace-instance level and per-repo level file ops
    2. For workspace-instance level: iterate over each stack in `stacks` map, call `mergeFiles(stack.files, workspace.files)`, then for each entry in copy and symlink arrays:
       - If `isGlobPattern(entry)`: call `expandGlob(entry, wsInstanceRoot)`, each match destination is `join(wsInstanceRoot, match)`
       - If literal: resolve via `resolveSourcePath(entry, wsInstanceRoot)` is irrelevant for destination; the destination is `join(wsInstanceRoot, isAbsolute(expandHome(entry)) ? basename(expandHome(entry)) : entry)` — exactly mirroring the logic in `processFileList`
    3. For per-repo level: iterate over `workspace.repos` where `mode === "worktree"`, find the matching stack repo from `stacks.get(wsRepo.stack)?.repos.find(r => r.name === wsRepo.name)`, call `mergeFiles(stackRepo?.files, wsRepo.files)`, then for each entry the destination is computed against `wsRepo.task_path` as destDir using the same logic
    4. For each computed destination path, check: does it start with `wsInstanceRoot + "/"` OR does it start with some `repo.task_path + "/"`? If not, it's external.
       - Simpler check: use `dest.startsWith(join(tasksDir, workspace.name) + "/")` as the internal boundary — anything under the workspace's task directory is internal.
       - Also check `dest === join(tasksDir, workspace.name)` for exact match.
    5. Collect external destinations and return as `Warning: external destination ${dest} was not removed` strings, one per destination.

    **Import additions to files.ts:**
    - Add `import type { Workspace, Stack } from "./config"` (already partially imported)
    - Add `import { expandHome } from "./paths"` (already imported)

    The function must NOT check whether the path exists (Pitfall 4 from research). It reasons from path math only.

    **Test file:** Add a new `describe("warnExternalFiles", ...)` block at the end of `tests/lib/files.test.ts`. Use the existing `makeTmpDir`/`cleanup` pattern. Create minimal Workspace and Stack objects with `files` config that produces both internal and external destinations.
