# Phase 01.1: File and Folder Copy/Symlink Support Verification Report

**Phase Goal:** Stack and workspace configs support a `files:` block with `copy` and `symlink` arrays that handle files, folders, and glob patterns at two independent levels (workspace-instance and per-repo), with loud-fail error semantics and idempotent re-application on both `new` and `open`

**Re-verification:** Yes — after gap closure

## Re-verification Focus

Previous verification found one gap (truth #20 partial): `workspace-wizard.ts` STEP 3 called `writeFileSync` directly without a symlink guard, so the `new` flow could write through a symlink at the env file path.

The fix applied an inline symlink guard in STEP 3 (lines 239-244 of `workspace-wizard.ts`):

```typescript
try {
  if (lstatSync(targetPath).isSymbolicLink()) {
    p.log.warn(`skipping env file write: ${targetPath} is a symlink`)
    continue
  }
} catch { /* file doesn't exist yet, safe to write */ }
writeFileSync(targetPath, content, "utf-8")
```

This uses `lstatSync` + `isSymbolicLink()` in a try/catch, semantically identical to the guard in `writeEnvFiles` (workspace-ops.ts lines 133-141). The behavioral truth is now satisfied in both the `new` and `open` flows.
