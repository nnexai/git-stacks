## Decisions Made

- **lstatSync not existsSync for destination check:** `existsSync` returns false for dangling symlinks; `lstatSync` in try/catch returns true for any path (file, dir, or broken symlink). Using lstatSync ensures idempotent re-application works even when symlink targets are missing.
- **Zero-match glob → warning not error:** A glob pattern matching zero files is common when configs reference optional files. Result type upgraded to `{ ok: true; warnings?: string[] }` so callers get visibility without aborting the operation.
- **Deprecated applyFileOperations kept:** The existing call site in workspace-wizard.ts uses the old signature. Keeping the deprecated shim prevents Plan 01 from breaking anything — Plan 02 will replace the call site.
- **Absolute/home-relative source → basename in dest:** When the source entry is absolute or starts with `~/`, the destination uses `basename(src)` to flatten it into the destDir. Relative entries preserve their full relative path structure in the dest.
