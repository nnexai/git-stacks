export function formatError(message: string, hint?: string): string {

Per CONTEXT.md locked decision: "Git operation failures: translate to a human-readable message, include the raw git error in parentheses for debugging." Reformat error return strings in workspace-ops.ts from colon-separated format to parenthetical format.

**cleanWorkspace error strings:**

1. `"pre_remove hook failed: ${err}"` ->
   `"pre_remove hook failed (${err})"`

2. Failure entries built as `\`${repo.name}: ${err}\`` ->
   `\`${repo.name} (${err})\``
   And the wrapper: `"Failed to clean worktrees:\n  ${failures.join("\n  ")}"` ->
   `"Could not clean worktrees:\n  ${failures.join("\n  ")}"`

**removeWorkspace error strings:**

3. `"pre_remove hook failed: ${err}"` ->
   `"pre_remove hook failed (${err})"`

4. Failure entries built as `\`${repo.name}: ${err}\`` ->
   `\`${repo.name} (${err})\``
   And the wrapper: `"Failed to remove worktrees:\n  ${failures.join("\n  ")}"` ->
   `"Could not remove worktrees:\n  ${failures.join("\n  ")}"`

**mergeWorkspace error strings:**

5. `"pre_remove hook failed: ${err}"` ->
   `"pre_remove hook failed (${err})"`

6. `"Merge failed for ${repo.name}: ${result.error}"` ->
   `"Merge failed for '${repo.name}' (${result.error})"`
   This is the key git operation error — result.error contains the raw git stderr.

7. `"Merge conflicts:\n  ${conflicting.join("\n  ")}"` — the `conflicting` entries are built as `\`${repo.name}: ${conflicts.join(", ")}\`` ->
   Change entries to: `\`${repo.name} (${conflicts.join(", ")})\``
   And the wrapper: `"Merge conflicts:\n  ${conflicting.join("\n  ")}"` ->
   `"Merge conflicts detected:\n  ${conflicting.join("\n  ")}"`

**syncWorkspace error strings:**

8. `"Conflicts detected:\n${detail}\n\nUse --best-effort to skip conflicting repos."` — the `detail` entries are built as `"  ${c.repo}: ${c.files.join(", ")}"` ->
   Change entries to: `"  ${c.repo} (${c.files.join(", ")})"`

**Concrete before/after example for the most important case (mergeWorkspace git failure):**

Before:
```typescript
return { ok: false, error: `Merge failed for ${repo.name}: ${result.error}` }
```

After:
```typescript
return { ok: false, error: `Merge failed for '${repo.name}' (${result.error})` }
```

**Do NOT change these error strings** (they are not git operation errors, they are config/state lookup errors and are already clear):
- `"Workspace '${name}' not found."` — config lookup, not a git error
- `"Dirty worktrees: ${dirty.join(", ")}"` — state check, not a raw git error
- `"Workspace '${newName}' already exists."` — config lookup
