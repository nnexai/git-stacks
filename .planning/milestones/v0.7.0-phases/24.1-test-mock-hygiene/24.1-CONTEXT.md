# Phase 24.1: Test Mock Hygiene

## Why This Phase Exists

Phase 24 migrated all 15 production files from `import * as p from "@clack/prompts"` to `import { prompts as p } from "@/tui/utils"`. It also added `_exec` injection to tmux.ts, cmux.ts, and lifecycle.ts.

However, the **test files** were not updated to match. This leaves:
- 7 test files with dead `mock.module("@clack/prompts")` calls that no longer intercept anything (production code doesn't import it)
- 1 test file (`artifacts.test.ts`) with incomplete `@/tui/utils` mock (missing `prompts` object — same bug we already fixed in `wizard-helpers.test.ts`)

## Concrete Scope

### Category 1: Remove dead `mock.module("@clack/prompts")` (7 files)

These files mock `@clack/prompts` but production code no longer imports it directly:

| File | Also mocks `@/tui/utils`? | Risk |
|------|---------------------------|------|
| `tests/tui/workspace-wizard.test.ts` | Yes (includes `prompts`) | Low — already has correct mock |
| `tests/commands/workspace-edit.test.ts` | Yes (includes `prompts`) | Low — already has correct mock |
| `tests/lib/integration-commands.test.ts` | No | Low — tested code never calls prompts |
| `tests/lib/integrations/wizard-helpers.test.ts` | Yes (just fixed) | None — already fixed |
| `tests/lib/integrations/niri.test.ts` | No | Medium — needs explicit `@/tui/utils` mock with `prompts` |
| `tests/lib/integrations/tmux.test.ts` | No | Medium — needs explicit `@/tui/utils` mock with `prompts` |
| `tests/lib/integrations/artifacts.test.ts` | Yes (incomplete — missing `prompts`) | HIGH — latent bug |

**Action:** Remove dead `@clack/prompts` mocks. For `niri.test.ts` and `integrations/tmux.test.ts`, add a proper `@/tui/utils` mock with `prompts` object (their `p.spinner()` calls currently work via fragile implicit mock-cache ordering). For `integration-commands.test.ts`, no replacement mock needed (only tests command structure, no prompt paths exercised).

### Category 2: Fix incomplete `@/tui/utils` mocks (1 file)

- `tests/lib/integrations/artifacts.test.ts` — mocks `@/tui/utils` with only `safeText`, missing `prompts` object

**Action:** Add `prompts` object to mock, same pattern as wizard-helpers fix.

## Out of Scope

- Migrating consumer tests from `mock.module("@/lib/tmux")` to `_exec` injection — these are valid integration-level mocks, not stale
- Dashboard test files that mock `@/lib/lifecycle` — these mock the entire module (valid)
- Any production code changes
