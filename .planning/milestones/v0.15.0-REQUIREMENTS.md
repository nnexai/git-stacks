# Requirements: git-stacks v0.15.0

**Defined:** 2026-04-04
**Core Value:** One command takes you from "I need to work on feature X" to a fully running dev environment — right repos, right branches, right IDE/terminal open, hooks run — without manual steps.

## v0.15.0 Requirements

Requirements for the Dir Mode & Polish milestone.

### Schema

- [x] **SCHM-01**: User can set repo type to "dir" in registry and template YAML
- [x] **SCHM-02**: Workspace YAML stores dir repos with mode "dir" and resolves main_path only (no task_path, no branch)

### Registry

- [x] **REG-01**: User can add a non-git directory to the repo registry via `repo add`
- [x] **REG-02**: `repo scan` detects and offers to register plain directories alongside git repos

### Workspace Lifecycle

- [x] **LIFE-01**: `git-stacks new` includes dir repos from template — references main_path directly, creates no worktree or branch
- [x] **LIFE-02**: `git-stacks open` includes dir repos in env/hook context but skips git operations
- [x] **LIFE-03**: `git-stacks close`/`clean`/`remove` skip worktree deletion for dir repos (nothing to delete)

### Git Operations

- [x] **GIT-01**: `git-stacks push` skips dir repos entirely
- [x] **GIT-02**: `git-stacks pull` skips dir repos entirely
- [x] **GIT-03**: `git-stacks sync` skips dir repos entirely
- [x] **GIT-04**: `git-stacks merge` skips dir repos (no branch to merge)
- [x] **GIT-05**: Ahead/behind tracking skips dir repos (no git state)
- [x] **GIT-06**: Dirty file detection skips dir repos

### Status & Display

- [x] **DISP-01**: `git-stacks status` shows dir repos with a "dir" label and no git metrics
- [x] **DISP-02**: `git-stacks list` includes workspaces with dir repos without git aggregation errors
- [x] **DISP-03**: TUI dashboard shows dir repos in workspace detail with "dir" indicator, no git badges

### Health

- [x] **HLTH-01**: `git-stacks doctor` skips git health checks for dir repos
- [x] **HLTH-02**: `git-stacks doctor` validates that dir repo paths exist and are accessible directories

## Future Requirements

Deferred to later in v0.15.0 or beyond — will be added after initial dir mode usage.

- Additional polish and fixes TBD

## Out of Scope

| Feature | Reason |
|---------|--------|
| Dir repos participating in git operations | Contradicts the purpose — dirs are non-git by definition |
| Dir repos creating worktrees | No .git to worktree from |
| Automatic dir content syncing | Not a git-stacks concern; use rsync/hooks if needed |
