### Gaps Summary

One gap blocks a clean combined test run: `tests/lib/integrations/wizard-helpers.test.ts` uses a minimal `@clack/prompts` mock that only includes the three properties it directly uses (`confirm`, `multiselect`, `isCancel`). The `workspace-wizard.test.ts` test file, which typically runs before it alphabetically in combined runs, registers a comprehensive mock with additional properties (`intro`, `outro`, `log`, `spinner`, `text`, `select`, `cancel`). Because Bun caches module bindings across test files in the same process, `wizard-helpers.ts` gets bound to the comprehensive mock's function instances rather than the wizard-helpers test's instances. The `beforeEach` resets in wizard-helpers.test.ts then reset the wrong functions, causing 4 tests to fail.

The fix is to expand the `@clack/prompts` mock in `wizard-helpers.test.ts` to include the same comprehensive set of properties that `workspace-wizard.test.ts` registers, following the same "comprehensive mock for shared modules" pattern documented in the Plan 02 summary.

This gap does not affect any runtime behavior — all production code is correct and passes typecheck. It is a test reliability issue only.
