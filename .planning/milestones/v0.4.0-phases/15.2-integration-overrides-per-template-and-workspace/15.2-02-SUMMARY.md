## Decisions Made

- **Co-location of runWorkspaceEdit**: Placed in workspace-wizard.ts rather than a new file, since it needs identical imports (config, clack, integrations, wizard-helpers). Keeps related wizard logic together.
- **Comprehensive @clack/prompts mock**: Both test files include spinner, multiselect, confirm, and other prompts in their mock to prevent Bun module cache collisions when the tests run in the same process. workspace-edit.test.ts runs before workspace-wizard.test.ts alphabetically; if it only mocked what it directly uses, workspace-wizard.ts would get bound to an incomplete mock.
- **settings strip in clone**: Used `const { cmux_workspace_id: _, settings: _existingSettings, ...restNoSettings } = source` to strip both cmux_workspace_id and settings before spreading, then explicitly reconstruct settings with merged integrations. This prevents the old settings from leaking into the new workspace alongside the new settings.

## Issues Encountered

**Bun module cache collision in combined test run**: When running both test files together (`bun test tests/tui/workspace-wizard.test.ts tests/commands/workspace-edit.test.ts`), workspace-wizard.ts was imported by workspace-edit.test.ts first with a minimal `@clack/prompts` mock (no spinner). Then when workspace-wizard.test.ts ran, workspace-wizard.ts was already cached with the incomplete binding, causing `p.spinner is not a function`.

**Fix (Rule 1 - Bug)**: Added comprehensive `@clack/prompts` mock to workspace-edit.test.ts (spinner, multiselect, confirm, log) so the first mock binding that workspace-wizard.ts receives includes all required properties. Both files now pass individually and together. This is the "comprehensive mock for shared modules" pattern established in Phase 15 (STATE.md).
