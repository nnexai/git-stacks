### Human Verification Required

#### 1. Background Dimming Visual Confirmation

**Test:** Run `bun run dev`, then `git-stacks manage` to launch the dashboard. In the Workspaces tab, press Enter on any workspace entry to open the action menu. Observe the area outside the centered dialog box.

**Expected:** The background behind the dialog appears visually distinct from the dialog itself — either a clearly dark/empty terminal background, or any dim effect that makes the dialog stand out as a modal overlay. The dialog should not visually "blend into" surrounding content.

**Why human:** `CenteredDialog` uses `backgroundColor="transparent"` on the outer full-screen box, while the PLAN specified `#1a1a1a` dark gray (D-08). Since the split pane is hidden when dialogs are active (not rendered underneath), the background is the terminal's default background, not a gray dim overlay. Whether this constitutes "dimming" per D-08 is a visual judgment call. The Task 3 visual verification checkpoint in Plan 03 was approved by the user — this check confirms the earlier approval still holds.

A final human confirmation closes this item.
