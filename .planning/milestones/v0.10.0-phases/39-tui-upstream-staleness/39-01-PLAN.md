  **Type definition (export it):**
  ```typescript
  export type StaleInfo = {
    count: number | null   // null = still loading
    error: boolean         // true = network failure (show ?)
    fetchedAt: number      // Date.now() timestamp
  }
  ```

  **Hook: `useStaleness()`**

  State:
  - `createSignal<Map<string, StaleInfo>>(new Map())` — cache keyed by `main_path`

  Constants:
  - `const STALENESS_TTL = 5 * 60 * 1000` (5 minutes, per D-07)

  Exported API:
  - `staleness(): Map<string, StaleInfo>` — the reactive signal accessor
  - `fetchStaleness(workspace: Workspace): void` — triggers async fetch for workspace repos if TTL expired
  - `invalidateCache(): void` — clears all `fetchedAt` to 0 so next `fetchStaleness` re-fetches (per D-11, STALE-03)

  **`fetchStaleness` implementation:**
  1. Get workspace repos. Deduplicate by `main_path` (D-13 — collect unique main_paths with a Set).
  2. For each unique `main_path`, check cache: if entry exists and `Date.now() - fetchedAt < STALENESS_TTL`, skip (D-10 — show cached data instantly).
  3. For repos needing fetch: immediately set `{ count: null, error: false, fetchedAt: 0 }` in the cache signal (D-05 — shows loading `...`).
  4. Then for each repo needing fetch, run async (use `Promise.allSettled` for concurrency, per research):
     a. Determine the branch to check:
        - Worktree repos: use `workspace.branch`
        - Trunk repos: use `getCurrentBranch(repo.main_path)` (current branch of main clone)
     b. Check `hasUpstreamTracking(repo.main_path, branch)` — if false, set `{ count: null, error: false, fetchedAt: Date.now() }` and continue (D-03/STALE-04 — no tracking means no badge)
     c. Wrap `fetchOrigin(repo.main_path)` in try/catch — on error, set `{ count: null, error: true, fetchedAt: Date.now() }` and continue (D-14/STALE-05 — network failure shows `?`)
     d. On successful fetch: call `getCommitsBehind(repo.main_path, "origin/" + branch, branch)` to get count
     e. Set `{ count, error: false, fetchedAt: Date.now() }` in cache
  5. Update the cache signal once after all fetches complete (or incrementally per repo for responsiveness — use batch signal update at end).

  **`invalidateCache` implementation:**
  Set all entries' `fetchedAt` to 0 so next `fetchStaleness` call treats them as expired.

  **Import only what's needed from `../../lib/git`:** `fetchOrigin`, `getCommitsBehind`, `hasUpstreamTracking`, `getCurrentBranch`.
  Import `createSignal` from `solid-js`.
  Import `type Workspace` from `../../../lib/config`.

  Follow existing hook patterns: no `onCleanup` needed (no timers or intervals — fetch is on-demand only, per D-12).
  </action>

  function resolveBadge(info: StaleInfo | undefined): { text: string; fg: string } {
    if (!info) return { text: "", fg: "gray" }                           // no data yet
    if (info.error) return { text: "?", fg: "red" }                     // D-04: network error
    if (info.count === null && !info.error && info.fetchedAt === 0)

    if (info.count === null) return { text: "", fg: "gray" }            // D-03: no upstream tracking
    if (info.count === 0) return { text: "", fg: "gray" }               // D-02: up to date, no badge
    return { text: `${info.count} behind`, fg: "yellow" }               // D-01: N behind in yellow
  }
