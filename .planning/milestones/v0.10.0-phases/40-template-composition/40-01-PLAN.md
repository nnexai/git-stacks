  constructor(public cycle: string[]) {
    super(`Circular includes detected: ${cycle.join(" -> ")}`)
  }
}

export class MissingTemplateError extends Error {
  constructor(public templateName: string, public referencedBy: string) {
    super(`Template '${templateName}' not found (referenced by '${referencedBy}' includes)`)
  }
}

**`composeTemplates(templateNames: string[]): Template`**

This function takes an ordered list of template names and produces a single merged Template. The last name in the array is the "top-level" template whose values take precedence.

Algorithm:
1. Load all templates via `readTemplate(name)`. If any name doesn't exist, throw `MissingTemplateError`.
2. For each template (in order), check its `includes` field. Load included templates and prepend them to the merge list (but do NOT recursively resolve their includes -- 1-level limit per D-12). If an included template itself has `includes`, log a warning to stderr: `[git-stacks] Warning: '${name}' has includes but nested includes are ignored (1-level limit)`.
3. Build the final ordered list: `[...all included templates in order, ...explicitly named templates in order]`. The last template in this list has highest precedence.
4. Check for circular references: build a visited set while resolving includes. If a template name appears twice, throw `CircularIncludesError` with the cycle path.
5. Merge in order (later overwrites earlier):
   - **repos:** Union by `repo` field (registry name). If same repo appears in multiple templates, keep the entry from the higher-precedence template BUT apply mode precedence: if ANY template had `mode: "worktree"` for that repo, the result is `"worktree"` (per D-04). For `base_branch` and `branch_pattern`, take from highest-precedence template that defines them.
   - **hooks:** For each hook key (`pre_create`, `post_create`, etc.), concatenate arrays in template order. The top-level template's hooks run last (per D-05).
   - **env:** `Object.assign()` in template order -- later keys overwrite earlier (per D-06).
   - **env_file:** Last non-undefined value wins.
   - **files.copy:** Concatenate arrays in template order. files.symlink: same.
   - **integrations:** Deep merge using `Object.assign()` per key in template order (per D-08).
   - **name:** Use the top-level template's name.
   - **description:** Use the top-level template's description.
   - **schema_version:** Use the top-level template's schema_version.
