Create `scripts/test-runner.ts` per D-01 through D-04. The script:

1. **File discovery** — Recursively scan `tests/` directory. Classify files per D-02:
   - Unit tests (run together in single `bun test` process):
     - `tests/lib/*.test.ts` (all 19 files)
     - `tests/lib/integrations/*.test.ts` (all 12 files)
     - `tests/tui/dashboard/*.test.tsx` where filename does NOT start with `integ-` (11 component test files)
     - `tests/tui/dashboard/snapshots/*.snap.test.tsx` (8 snapshot test files)
     - `tests/tui/dashboard/configUtils.test.ts` (1 file)
   - Integration tests (run per-file in isolated processes):
     - `tests/commands/*.test.ts` (7 files)
     - `tests/tui/*.test.ts` (2 files: `messageUtils.test.ts`, `workspace-wizard.test.ts`)
     - `tests/tui/dashboard/integ-*.test.tsx` (4 files)

2. **Execution modes** — Accept an optional CLI argument:
   - No argument or `--all`: Run both unit and integ
   - `--unit`: Run unit tests only
   - `--integ`: Run integ tests only

3. **Unit test execution** — Single `Bun.spawn(["bun", "test", ...unitFiles])` call. Set `cwd` to project root so `bunfig.toml` preload applies. Pipe stdout/stderr to parent process via `stdio: ["inherit", "inherit", "inherit"]`.

4. **Integration test execution** — For each integ test file, spawn `Bun.spawn(["bun", "test", filePath])` individually. Run sequentially (simpler, avoids interleaved output). Set `cwd` to project root. Before each file, print a header line: `\n--- [integ] ${relative(ROOT, filePath)} ---`. Pipe stdio.

5. **Result aggregation** — Track pass/fail count for each spawned process. Print summary at end:
   ```
   === Test Results ===
   Unit tests: PASS (or FAIL)
   Integration tests: X/Y passed
   ```
   Exit with code 0 only if ALL processes exited 0; otherwise exit 1.

6. **Update `package.json`** scripts per D-03:
   ```json
   "test": "bun run scripts/test-runner.ts",
   "test:unit": "bun run scripts/test-runner.ts --unit",
   "test:integ": "bun run scripts/test-runner.ts --integ",
   ```
   Keep `prepublishOnly` as is (it calls `bun test` but now `bun run test` is the custom runner).
   Actually update `prepublishOnly` to: `"prepublishOnly": "bun run typecheck && bun run test"` (use `bun run test` instead of `bun test` to go through the runner).

IMPORTANT: The runner file must be executable TypeScript run by Bun. No shebang needed since it's invoked via `bun run scripts/test-runner.ts`. Use `import.meta.dir` to compute ROOT as `join(import.meta.dir, "..")`. Use `readdirSync` with `withFileTypes: true` for recursive file discovery. Use `Bun.spawn` (not `child_process`).
