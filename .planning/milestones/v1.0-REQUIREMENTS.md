## v1 Requirements — All Satisfied

### Foundation — Tests

- [x] **TEST-01**: Developer can add git integration tests using a `makeGitRepo(tmpDir)` helper that creates a real local git repo in a temp directory — **SATISFIED** (Phase 01-01)
- [x] **TEST-02**: Core git operations (createWorktree, removeWorktree, mergeNoFF, rebaseBranch, getCommitsBehind) have integration test coverage with real git repos — **SATISFIED** (Phase 01-01)
- [x] **TEST-03**: Workspace lifecycle critical paths (open, clean, remove, merge) have integration tests that catch the known partial-failure bugs — **SATISFIED** (Phase 01-05)

### Foundation — Config Robustness

- [x] **CONF-01**: A corrupt or outdated YAML file in `workspaces/` or `stacks/` does not crash `list` or any other command — it logs a warning to stderr and skips that entry — **SATISFIED** (Phase 01-02)
- [x] **CONF-02**: All new schema fields use `.optional()` or `.default()` — no new required fields can break existing user config files on upgrade — **SATISFIED** (Phase 01-02)
- [x] **CONF-03**: A `schema_version` field is added to Stack and Workspace YAML schemas, with a documented migration strategy for future breaking changes — **SATISFIED** (Phase 01-02)
- [x] **CONF-04**: Zod validation errors surface as human-readable field-level messages (e.g., "repos[0].path: expected string, got undefined") not raw stack traces — **SATISFIED** (Phase 01-02)

### Foundation — Prerequisites

- [x] **PREREQ-01**: On startup, git is verified to be installed and meets the minimum required version (2.24+ for worktree support); a clear error is shown if not met, with install instructions — **SATISFIED** (Phase 01-03)
- [x] **PREREQ-02**: Each integration checks for its required binary at `applies()` time before generating artifacts or launching — missing binary causes that integration to be skipped with a debug-level note — **SATISFIED** (Phase 01-03)
- [x] **PREREQ-03**: `doctor` reports missing runtime dependencies (git version, integration binaries: `code`, `code-insiders`, `idea`, `tmux`, `cmux`) with suggested install commands — **SATISFIED** (Phase 01-03)

### Foundation — Bug Fixes

- [x] **BUG-01**: `mergeWorkspace` is atomic — the workspace YAML is not deleted until all repo merges succeed — **SATISFIED** (Phase 01-04)
- [x] **BUG-02**: `removeWorkspace` and `cleanWorkspace` use a stage-then-commit pattern — state is not permanently mutated until all operations succeed — **SATISFIED** (Phase 01-04)
- [x] **BUG-03**: `renameWorkspace` re-registers git worktrees at their new paths via `git worktree remove` + `git worktree add` — **SATISFIED** (Phase 01-04)
- [x] **BUG-04**: A failed `mergeNoFF` does not leave the main clone stranded on the base branch — **SATISFIED** (Phase 01-04)

### Version Information

- [x] **VER-01**: `git-stacks -V` / `--version` shows the version from `package.json` plus the git commit hash (with `-dirty` suffix when working tree has uncommitted changes) when running from source — **SATISFIED** (Phase 01.2-01)

### File Operations — Copy/Symlink Support (Phase 01.1)

- [x] **FILES-01**: `copy` operation copies a single file from source to destination — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-02**: `copy` operation copies a folder tree recursively using `cpSync` with `recursive: true` — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-03**: `symlink` operation creates a file symlink from destination pointing to source — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-04**: `symlink` operation creates a directory symlink from destination pointing to source — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-05**: Glob pattern `.env.*` expands to matching dotfiles (requires `dot: true` in `Bun.Glob`) — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-06**: Glob pattern `secrets/**` expands to all files under the matched directory — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-07**: A glob pattern matching zero files produces a warning (not an error) — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-08**: When destination already exists, the operation is skipped silently — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-09**: When destination is missing and source is missing, a loud error is returned — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-10**: When destination is missing and source exists, the operation is applied — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-11**: Source paths starting with `~/` are expanded via `expandHome()` from `paths.ts` — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-12**: Absolute source paths are used as-is — **SATISFIED** (Phase 01.1-01)
- [x] **FILES-13**: Workspace-instance level `files:` applies to the workspace root directory — **SATISFIED** (Phase 01.1-02)
- [x] **FILES-14**: Per-repo level `files:` resolves relative source paths against the repo's `main_path` and applies to `task_path` — **SATISFIED** (Phase 01.1-02)
- [x] **FILES-15**: Stack `files:` and workspace `files:` merge additively — **SATISFIED** (Phase 01.1-02)
